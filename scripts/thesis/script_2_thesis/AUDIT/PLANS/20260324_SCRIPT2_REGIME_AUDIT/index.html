    Script 2 Regime Audit

    Scope

    • Baseline run under analysis: results/20260319_035820
    • Rescue run under analysis: results/20260323_tierA_rescue_v2
    • Audit log: 20260323_SCRIPT2_AUDIT_LOG.md
    • Tier A rescue audit: 20260323_SCRIPT2_TIER_A_RESCUE_AUDIT.md
    • Thesis plan reference: EXPERIMENT_PLAN_THESIS_VALIDATION.md
    • Status note: AXV-F1 from the rescue run is still executing, so axial conclusions remain provisional.

    Executive Summary

    • Tier A does not validate the trading thesis economically yet. There is weak learnability, but absolute edge is still tiny.
    • After unifying baseline Tier A with Tier A rescue, only two trainable best trials are positive:
    • ViT-F1: PnL=+81.52, Return=+0.008%
    • MLP-F3: PnL=+16.01, Return=+0.002%
    • MLP is the family that benefited the most from rescue. The rescue improved search quality materially, but it did not convert Tier A into a robust money-making tier.
    • TCN shows repeated one-sided collapse behavior. In Tier A it looks more like a degeneration pattern than a trustworthy edge source.
    • ViT can survive when it becomes extremely selective. The best surviving case is not broad trading skill. It is disciplined abstention.
    • The current 3 x 3 x 3 = 27 combined regime label is too sparse for first operational use. In validation, the dominant combined state only explains about 8% to 12% of bars, so "same top combined regime" is not enough by itself.
    • Tier B remains the correct next priority, but the next tiers should be analyzed and likely redesigned with regime-aware logic.

    Method Notes

    • Fold windows come from the Script 2 generator and thesis plan.
    • Effective validation windows were not estimated manually. They were read from prediction parquet metadata in the database.
    • Regime classification in this audit uses the current legacy regime logic:
    • trend by ADX
    • volatility by ATR ratio versus rolling average
    • direction by DI +/- and EMA fast/slow
    • "Unified Tier A + rescue" means:
    • for each model/fold pair, compare baseline best trial and rescue best trial
    • keep the stronger result as the current best known state

    Why The Current Combined Regime Is Not Enough

    • The top combined regime in validation only carries:
    • F1: 9.1%
    • F2: 7.9%
    • F3: 11.1%
    • F4: 12.4%
    • That means almost 88% to 92% of validation bars live outside the dominant combined state.
    • Consequence:
    • even when train and validation share the same top combined state, most of the fold is still governed by other states
    • the practical regime distribution can drift while the top label appears unchanged

    • MLP is the most promising family in Tier A after rescue.
    • TCN still looks structurally fragile.
    • ViT remains the best candidate for selective trading under weak features.
    • axial_vit is still unresolved.

    • The strongest criticism of Tier A is simple:
    • many best-objective trials are still money losers
    • This means the current objective can still reward predictive quality that does not translate into enough economic edge.
    • Tier A is therefore useful as a scientific probe, but not yet as a deployment-grade signal source.

    What Likely Happened

    1. Tier A is too poor in information

    • close + RSI is enough to reveal learnability differences between families
    • it is not enough to produce robust, portable edge across these folds
    • this is the strongest argument for continuing with Tier B

    2. Selectivity beats activity in weak-feature space

    • the only positive unified cases are the most selective ones
    • that suggests the best Tier A use may be filtering or abstention logic, not a full standalone signal engine

    3. Rescue improved search more than it improved the thesis

    • rescue found better pockets, especially for MLP
    • but most improved pockets still remain economically negative
    • this means the rescue was useful operationally, but it does not rescue the business conclusion by itself

    4. Same regime name is not enough

    • F3 and F4 prove this from opposite angles:
    • F3: same top combined state, one family wins
    • F4: same top combined state, nobody wins
    • the regime state has to be treated as a distribution, not as a single winner label

    5. The objective may still be too tolerant of unprofitable behavior

    • best-objective does not consistently imply best-money
    • for future tiers, this should be monitored aggressively
    • this is not yet a call to rewrite the objective, but it is a clear warning sign

    Strategic Implications for Tier B

    • Continue with Tier B.
    • Treat Tier A as a low-information scout, not as the final battlefield.
    • Prioritize MLP and ViT as the most interesting families to carry forward.
    • Treat TCN with skepticism until it proves it can stop collapsing into one-sided behavior.
    • Keep AXV in observation mode until the current rescue finishes and the family has enough completed evidence.

    Strategic Implications for Future Folds

    Strong Recommendation

    • Do not keep relying only on arbitrary rolling windows once Tier B is finished.

    Recommended Progression

    1. Start with a small regime taxonomy:
    2. first try 3 families or 9 families
    3. do not jump directly to 27 for production decisions
    4. Evaluate regime similarity between train and validation explicitly:
    5. trend distribution similarity
    6. volatility distribution similarity
    7. direction distribution similarity
    8. combined distribution similarity
    9. Build "same-regime validation" experiments:
    10. train on one regime family
    11. validate on the same regime family
    12. Only after that, evaluate a mixture-of-experts idea:
    13. detect current regime
    14. choose model family by regime

    Immediate Work Plan

    1. Refactor the old regime scripts into a shared scripts/analysis/regime/ core plus thin CLIs.
    2. Add one Script 2 specific regime analyzer that reads DB + parquets directly.
    3. Re-run this audit automatically once AXV-F1 rescue completes.
    4. Apply the same framework to Tier B as soon as enough runs exist.
    5. Compare 3, 9, and 27 regime taxonomies on coverage, stability, and practical usefulness.
    6. Design candidate regime-based folds after Tier B conclusions are in.

    Current Thesis Reading

    • The system is not dead.
    • The system is not validated yet.
    • There is evidence of learnability.
    • There is not yet evidence of robust autonomous profitability from Tier A.
    • The most honest reading is:
    • keep going
    • move to richer features
    • make regime analysis first-class
    • stop pretending that a single temporal fold design is enough
