    Script 2 Ensemble Phase 1 Block B Audit

    • Date: 2026-04-04
    • Scope: Preliminary audit of the cross-model ensemble monetization block
    • Batch: 20260403_phase1_block_b
    • Status: Preliminary and materially informative

    1. Purpose

    This note records what the cross-model ensemble block now tells us about the main Script 2 ensemble question:

    Do cross-model combinations look stronger than single-family ensembles when the real objective is net monetization under friction?

    This is the most important Phase 1 question, because the thesis-side value of ensemble work is not merely “can one family vote with itself,” but whether different families can be combined into a better economic decision process.

    The short answer from Block B is:

    • yes, preliminarily
    • and the evidence is stronger than it was after Block A alone

    2. What Was Executed

    Block B is the cross-model monetization block defined in 20260403_SCRIPT2_ENSEMBLE_DISCOVERY_CAMPAIGN.md.

    It used:

    • Tier B source pool only
    • 20260325_tierB_mlp_ex
    • 20260328_tierB_corrected
    • folds:
    • F1
    • F3
    • cross-model pools:
    • tcn,vit
    • mlp,tcn
    • mlp,vit
    • mlp,tcn,vit
    • logistic,mlp,tcn,vit
    • ranking modes:
    • promise
    • friction
    • top_per_model = 1
    • diversity modes:
    • off
    • mild
    • tight
    • merge rules:
    • the current 8-rule battery

    Execution summary:

    • 60 completed plan executions
    • 480 rule-level ensemble runs
    • 28 / 60 executions produced a positive best ensemble

    The campaign-level annex is here:

    • 20260404_SCRIPT2_ENSEMBLE_PHASE1_BLOCK_B_CAMPAIGN_REPORT.md

    3. Why Block B Matters More Than Block A

    Block A already showed that ensembles could be economically interesting.

    But Block A still stayed inside one family at a time.

    Block B is more important because it asks whether the ensemble idea improves when we let different model families interact.

    That is much closer to the real thesis-side hypothesis:

    • different families may capture partially different edges
    • consensus or partial agreement between them may create a more selective and more economically useful signal

    Block B therefore deserves more weight than Block A when we choose what to do next.

    4. Main Positive Signals

    4.1 The strongest cross-model pool is now closer to breakeven than the best Block A family

    From Block A:

    • best family by average best-ensemble PnL was vit at -$17.32

    From Block B:

    • best cross-model pool by average best-ensemble PnL is mlp,tcn,vit at -$3.85

    That is still not a final profitability claim.

    But it is an important directional signal:

    • the best cross-model pool is now economically closer to neutral than the best single-family pool

    4.2 Fold F1 is now clearly positive on average

    Block B fold summary:

    • F1 average best-ensemble PnL: +$46.90
    • F3 average best-ensemble PnL: -$239.14

    This is stronger than the Block A picture, where:

    • F1 was only -$9.17 on average

    So the current evidence supports a more specific statement:

    • the cross-model thesis looks especially alive in F1

    4.3 Tight diversity again comes out best

    Just as in Block A, Block B again favors tight as the best diversity mode by average best-ensemble PnL:

    • tight: -$79.13
    • mild: -$104.48
    • off: -$104.76

    This is still not proof that the chosen thresholds are optimal.

    But it is now a repeated pattern across both Phase 1 blocks, which gives the diversity idea substantially more credibility.

    4.4 There are very strong better-than-proportional cases

    The best Block B execution:

    • block_b_mlp_tcn_vit_f1_friction_top1_tight
    • best net PnL: +$407.52
    • delta vs best source: +$499.05
    • Trade Red% = 64.62%
    • PnL Imp% = 545.22%
    • Selectivity Surplus = 480.59%
    • Gross/trade Chg% = 257.29%

    That is exactly the kind of result that argues against a simple “less trading, less friction” story.

    It says:

    • the ensemble traded much less
    • net improved dramatically more than the trade cut alone would suggest
    • and the economic quality of the remaining trades improved sharply

    5. Pool-Level Reading

    5.1 mlp,tcn,vit now looks like the strongest core candidate

    Among the tested pools, mlp,tcn,vit is the best by average best-ensemble PnL:

    • average best PnL: -$3.85
    • positive-execution rate: 58.33%

    That makes it the strongest current candidate for the “core money pool.”

    5.2 logistic,mlp,tcn,vit is interesting but not yet clean

    The broader pool with logistic is second by average best-ensemble PnL:

    • average best PnL: -$44.70
    • positive-execution rate: 58.33%

    So logistic cannot be dismissed.

    But the block also shows that this broader pool can produce unstable turnover patterns in some executions, especially on the gross side.

    That means:

    • logistic may add useful diversity
    • but it is not yet proven to be a universally stabilizing member

    5.3 mlp,tcn looks high-upside but unstable

    The mlp,tcn pair has:

    • the highest positive-execution rate: 66.67%

    But its average best-ensemble PnL is still:

    • -$173.47

    That usually means:

    • some very strong wins
    • but too much instability or too many weak executions around them

    So this pair should not be discarded, but it also should not yet be treated as the clean winner.

    6. Selectivity vs Friction Findings

    This remains the key interpretive lens.

    6.1 Better-than-proportional evidence remains strong

    Block B continues the same pattern already seen in Block A:

    • several top executions are clearly Better than proportional
    • the best cases improve much more than a simple trade cut would imply

    That matters because it supports the stronger interpretation:

    • at least some ensembles are selecting better opportunities
    • not merely suppressing turnover

    6.2 Gross-per-trade still matters as confirmation

    Block B family summary shows:

    • mlp,tcn,vit: gross-per-trade improvement rate 58.33%
    • logistic,mlp,tcn,vit: 58.33%
    • tcn,vit: 58.33%

    This is not overwhelming proof on its own.

    But together with the better-than-proportional cases, it strengthens the case that the cross-model ensembles are sometimes preserving or improving gross edge per decision.

    7. Merge-Rule Caveat Still Applies

    Block B is more informative than Block A for merge rules, but the same core warning still applies:

    • do not overread merge-rule rankings yet

    Why:

    • the pools are larger than Block A, but they are still small enough that some voting rules collapse toward similar behavior
    • all_must_agree and majority_vote:0.85 still look nearly identical in the aggregate tables
    • several strong outcomes are driven at least as much by the selected source pool as by the merge rule itself

    So the correct reading is:

    • merge-rule behavior is becoming more interesting
    • but the current evidence is still not a final verdict on merge design

    8. What Block B Changes In The Overall Picture

    After Block A alone, the ensemble direction looked promising.

    After Block B, the picture is stronger:

    1. the main cross-model thesis now looks more credible than it did after single-family testing alone
    2. the best cross-model pool is now stronger than the best single-family pool on average
    3. F1 now looks like a serious monetization zone for cross-model ensembles
    4. repeated support for tight diversity means the diversity-aware planner is no longer just an implementation feature; it is part of the economic story

    That is enough to justify continued expansion.

    9. What This Still Does Not Prove

    Even after Block B, the current evidence still does not prove:

    1. that ensembles are broadly profitable overall
    2. that one merge rule is the final winner
    3. that logistic should always be included
    4. that the current diversity thresholds are optimal
    5. that the same story will hold in later folds or under source-pool ablations

    So the signal is stronger now, but still preliminary.

    10. Recommended Next Block

    The next block should not be F2/F4 controls yet.

    The next block should be a source provenance ablation.

    Why this is the best next step:

    Block B already suggests that cross-model pools can work.

    The next practical question is:

    Are the good ensembles mainly coming from 20260325_tierB_mlp_ex, from 20260328_tierB_corrected, or only from mixing both?

    That is more actionable right now than jumping straight to harder control folds, because it tells us where the winning members are actually coming from.

    Recommended Phase 2 ablation:

    • source pools:
    • 20260325 only
    • 20260328 only
    • combined
    • model pools:
    • tcn,vit
    • mlp,tcn
    • mlp,tcn,vit
    • logistic,mlp,tcn,vit
    • folds:
    • F1
    • F3
    • ranking:
    • promise
    • friction
    • diversity:
    • off
    • tight

    That block should answer whether the current cross-model signal is:

    • coming mainly from one run
    • strengthened by mixing runs
    • or diluted by mixing incompatible frontier styles

    11. Bottom Line

    Block B is the strongest ensemble evidence so far.

    The correct takeaway is:

    • the ensemble direction is now more credible than it was after Block A
    • cross-model combinations appear more promising than single-family ensembles
    • the next intelligent step is to understand source provenance before expanding blindly
