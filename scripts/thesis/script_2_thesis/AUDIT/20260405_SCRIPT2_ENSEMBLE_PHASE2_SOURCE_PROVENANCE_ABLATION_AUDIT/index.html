    Script 2 Ensemble Phase 2 Source Provenance Ablation Audit

    • Date: 2026-04-05
    • Scope: Preliminary audit of the source-provenance ablation block
    • Batch: 20260404_phase2_source_provenance_ablation
    • Status: Preliminary, materially informative, and decision-shaping

    1. Purpose

    This note records what the source-provenance ablation block now tells us about the current Script 2 ensemble thesis:

    Are the strongest ensemble results coming mainly from one source frontier, or does the corrected Tier B pool contribute enough to justify mixing it in as a primary source?

    This question matters because Block B already suggested that cross-model ensembles can be economically interesting. The next uncertainty was whether the source frontier behind those ensembles should be:

    • 20260325_tierB_mlp_ex only
    • 20260328_tierB_corrected only
    • or both combined

    The short answer from this block is:

    • 20260325_tierB_mlp_ex is currently the strongest primary source pool
    • the combined pool is still interesting, but secondary
    • 20260328_tierB_corrected alone should be deprioritized for now

    2. What Was Executed

    This block used:

    • source provenance modes:
    • 20260325_tierB_mlp_ex
    • 20260328_tierB_corrected
    • both combined
    • folds:
    • F1
    • F3
    • model pools:
    • tcn,vit
    • mlp,tcn
    • mlp,tcn,vit
    • logistic,mlp,tcn,vit
    • ranking modes:
    • promise
    • friction
    • top_per_model = 1
    • diversity modes:
    • off
    • tight
    • merge rules:
    • the current 8-rule battery

    Execution summary:

    • 96 completed plan executions
    • 768 rule-level ensemble runs
    • 41 / 96 executions produced a positive best ensemble

    The campaign-level annex is here:

    • 20260405_SCRIPT2_ENSEMBLE_PHASE2_SOURCE_PROVENANCE_ABLATION_CAMPAIGN_REPORT.md

    3. Main Provenance Findings

    3.1 20260325_tierB_mlp_ex is the strongest primary source frontier

    Average best-ensemble net PnL by provenance:

    • 20260325_tierB_mlp_ex: -$24.08
    • 20260325_tierB_mlp_ex + 20260328_tierB_corrected: -$72.18
    • 20260328_tierB_corrected: -$290.02

    Positive best-ensemble rate by provenance:

    • 20260325_tierB_mlp_ex: 65.62%
    • combined: 53.12%
    • 20260328_tierB_corrected: 9.38%

    Selectivity surplus by provenance:

    • 20260325_tierB_mlp_ex: +84.34%
    • combined: +62.85%
    • 20260328_tierB_corrected: -31.95%

    This is the most important result of the block.

    It means the current ensemble edge is not being sourced evenly across the two Tier B frontiers. Right now, the auto-selected frontier is doing most of the economic heavy lifting.

    3.2 The combined pool is still viable, but not the leader

    The combined provenance is not a failure.

    It remains second on:

    • average best-ensemble PnL
    • positive-execution rate
    • average PnL delta vs best source

    It also still produces some of the strongest individual executions, especially in F1.

    So the combined pool should not be discarded.

    But this batch does not support treating the combined frontier as the new default primary source. It still trails the simpler 20260325-only setup on the metrics that matter most.

    3.3 20260328_tierB_corrected alone is weak enough to deprioritize

    20260328_tierB_corrected alone is materially worse than the other two source modes:

    • average best-ensemble PnL: -$290.02
    • positive best-ensemble rate: 9.38%
    • average selectivity surplus: -31.95%

    This does not prove that the corrected frontier has zero value.

    But it does say something important:

    • it is currently not competitive as a standalone primary ensemble source pool
    • if it contributes later, it is more likely as a selective diversity donor than as the main frontier

    4. Fold-Level Reading

    4.1 F1 remains the main money fold

    Average best-ensemble PnL by provenance inside F1:

    • 20260325_tierB_mlp_ex: +$114.86
    • combined: +$74.16
    • 20260328_tierB_corrected: -$83.78

    So F1 still carries the strongest evidence that the ensemble thesis can produce real monetization.

    4.2 F3 is still materially weaker

    Average best-ensemble PnL by provenance inside F3:

    • 20260325_tierB_mlp_ex: -$163.02
    • combined: -$218.52
    • 20260328_tierB_corrected: -$496.26

    That does not erase the positive F3 cases.

    But it does reinforce the current asymmetry:

    • F1 is the fold where the thesis looks alive
    • F3 still shows partial support, but not with the same strength

    5. Recipe-Level Reading

    5.1 The best cross-fold recipes are mostly promise-ranked

    When grouping by full recipe across F1 and F3 (source pool + model pool + rank mode + diversity mode), the strongest average net PnL results are:

    • combined logistic,mlp,tcn,vit, promise, tight: +$185.58
    • 20260325 only logistic,mlp,tcn,vit, promise, off: +$153.70
    • 20260325 only logistic,mlp,tcn,vit, promise, tight: +$147.96
    • 20260325 only tcn,vit, promise, tight: +$140.86
    • 20260325 only mlp,tcn,vit, promise, tight: +$140.25
    • 20260325 only mlp,tcn, promise, off: +$138.03

    Important caveat:

    • each of these recipe averages comes from only 2 executions (F1 and F3)

    So they are highly informative, but still preliminary.

    5.2 The biggest absolute winners are still concentrated in F1

    Top net-PnL execution:

    • phase2_prov_tierb_auto_only_mlp_tcn_vit_f1_friction_top1_off
    • best net PnL: +$426.42
    • delta vs best source: +$517.95

    This is again a strong better-than-proportional case and reinforces the idea that the ensemble edge is currently concentrated in F1, especially when the source frontier is 20260325-only.

    5.3 Merge-rule readings remain secondary

    This report still shows all_must_agree frequently landing on top.

    That is useful to note, but it should not be over-read yet.

    The current member counts are still modest, and several rules can behave similarly when the pool is small. The provenance result is much more important than the merge-rule ranking at this stage.

    6. What This Changes

    This block narrows the next search space significantly.

    Before this ablation, it was still reasonable to treat the two Tier B source frontiers as roughly interchangeable inputs.

    After this block, that is no longer the best reading.

    The current evidence says:

    • use 20260325_tierB_mlp_ex as the primary source frontier
    • keep the combined pool as a secondary candidate
    • stop spending major search budget on 20260328_tierB_corrected alone

    That is a meaningful simplification.

    It reduces experimental branching while staying aligned with the evidence.

    7. Recommended Next Step

    The next useful question is no longer source provenance.

    It is cross-fold transfer.

    More specifically:

    • take the strongest recipe families discovered across F1 and F3
    • and test those same recipes on F2 and F4

    That is the most direct way to answer the next thesis-side question:

    Can the best current ensemble recipes survive outside the discovery folds, or is the current edge too fold-local?

    Under the current planner workflow, this means:

    • replaying the same recipe specification on F2 and F4
    • using the same source pool, model pool, ranking mode, diversity mode, and top_per_model
    • while allowing the planner to re-select the actual fold-compatible trials

    So this is recipe transfer, not exact fixed-trial transfer.

    That is still useful and still worth doing now.

    If later we want exact fixed-trial transfer, the runner can already execute manual --source-trial-id sets, but the current batch workflow does not yet orchestrate that pattern.

    8. Three-Run Synthesis

    Taken together, the three completed ensemble blocks now give a much clearer picture than any single block alone.

    8.1 What looks materially supported now

    • the ensemble thesis is no longer just speculative
    • there are repeated better-than-proportional cases
    • diversity-aware selection, especially tight, keeps recurring as useful
    • F1 is the clearest money fold
    • cross-model pools are more important than single-family pools
    • the strongest current primary source frontier is 20260325_tierB_mlp_ex

    That is already a meaningful body of evidence.

    8.2 What pattern is emerging

    The current pattern is not random.

    It looks like this:

    1. the source frontier matters a lot
    2. the strongest frontier today is 20260325_tierB_mlp_ex
    3. cross-model combinations matter more than staying inside one family
    4. tighter diversity constraints often help rather than hurt
    5. the best outcomes often come from reducing trade count while improving economic quality per remaining trade

    That is a coherent methodological direction, not just a few lucky runs.

    8.3 What confidence we should claim

    I would describe the current confidence as:

    • high confidence that the ensemble path is worth pursuing
    • medium confidence that the best current direction is 20260325-primary, cross-model, diversity-aware
    • low confidence on universal merge-rule conclusions
    • low-to-medium confidence on fold generalization beyond F1

    So the big green light is on the ensemble direction itself, but not yet on a single final production recipe.

    9. Cross-Fold Methodology

    The next question is whether we can find one model or ensemble recipe that is positive across all four folds.

    That is a very good next target, but it has to be split into two different methodologies.

    9.1 Recipe transfer

    This is what the new Phase 3 batch does.

    It keeps the same recipe:

    • same source pool
    • same model pool
    • same ranking mode
    • same diversity mode
    • same top_per_model

    and re-runs the planner on F2 and F4.

    That means:

    • the recipe is transferred
    • the exact selected trials are not fixed

    This is the easiest next test and still very informative.

    9.2 Fixed-trial transfer

    This is the harder and stricter test.

    It would mean:

    • choose one exact set of source_trial_ids
    • run those exact same trained models on the other fold ranges

    This is not the same as planner-based transfer.

    It answers a different question:

    Do the exact trained members that looked good in one discovery fold still behave well on the other fold windows?

    9.3 Does the current backtest stack allow it?

    Yes, partially.

    The current runner already supports:

    • repeated --source-trial-id
    • manual --range-start
    • manual --range-end

    So the core execution path for fixed-trial transfer already exists.

    What the current batch workflow does not do yet is orchestrate that mode as a thesis block automatically. The current batch runner is planner-first and fold-first.

    9.4 Why this becomes four experiments, not one

    A true all-fold validation cannot be one single backtest.

    Each fold has its own execution range, so one fixed-trial cross-fold study would naturally become:

    • one run on F1
    • one run on F2
    • one run on F3
    • one run on F4

    Then their results should be aggregated afterward into one fold-level summary.

    That part is conceptually clean.

    It just needs a small orchestration layer above the existing runner.

    9.5 Do we need prediction regeneration?

    Most likely, yes.

    If the fixed trials were never executed on the target fold range, the needed prediction artifacts probably do not exist yet for that window.

    In that case, the current runner logic would:

    • try to reuse a compatible predictions parquet
    • and regenerate it when missing

    So fixed-trial cross-fold is not blocked by missing predictions.

    It is mostly a workflow/orchestration problem, not a core-engine problem.

    10. Conclusion

    This block is a strong narrowing step.

    It does not just say that ensembles can work.

    It says something more actionable:

    • the current edge is mostly coming from 20260325_tierB_mlp_ex
    • the combined pool is still relevant but not the leader
    • 20260328_tierB_corrected alone is currently too weak to prioritize
    • F1 remains the clearest monetization fold
    • the next smart step is to test whether the best recipes transfer into the harder folds

    That gives Phase 3 a clear direction without exploding the search space again.

    And after that, if the signal still holds, the next serious escalation is fixed-trial all-fold validation to see whether we can identify one ensemble recipe family that remains economically positive across all four folds.
