    2.3 The Friction Mechanism (Data-Driven)

    The mechanism is:

    1. A model with better S2/F1 detects more patterns → generates more trade signals
    2. More signals → more trades (the gate lets more through)

    3. Each trade costs friction (slippage + commission round-trip)

    4. Profitable trials average 106 trades

    5. Losing trials average 239 trades
    6. Trade difference: 133 extra trades
    7. Average friction per trade: $2.97
    8. Estimated friction drag from extra trades: $395

    When PnL is normalized per trade (removing the trade-count effect), positive correlations emerge — confirming that better models DO capture slightly more edge per trade, but not enough to offset the friction from trading more often.

    Friction Paradox Insight: The global S2-vs-PnL correlation is r=-0.012 (effectively zero), while Trades-vs-PnL is r=-0.813 (EXTREME negative). This is the core friction paradox: signal quality alone does not predict profitability because more signal → more trades → more friction. Implication for rescue: models must be MORE SELECTIVE (higher threshold, stricter gate) to reduce trade count and minimize friction drag.

    12. Key Takeaways for Tier B Rescue Design

    Data-driven synthesis from 1181 trials (≥5 trades) out of 1183 total completed:

    The Friction Paradox (CRITICAL)

    1. S2 vs PnL: r=-0.012 — Signal quality does NOT predict profitability
    2. Trades vs PnL: r=-0.813 — More trades STRONGLY predicts losses
    3. Profitable trials average 106 trades vs 239 for losing (Δ=133 trades)
    4. Only 1.2% of trials are profitable — selectivity is the key differentiator

    Overfitting Dynamics

    1. Feature count vs S1-S2 gap: r=+0.377 — More features = more overfitting risk
    2. 45 forced features require aggressive regularization to prevent memorization
    3. Weight_decay shows surprising positive correlation with gap (r=+0.206)

    Rescue Configuration Recommendations

    1. Test selectivity levers explicitly: higher prediction thresholds and entry-gate neighborhoods
    2. Entry gate weight vs Trades: r=-0.012, vs PnL: r=+0.002
    3. Keep regularization model-specific: search broader dropout/weight_decay, but avoid one blanket bias
    4. Increase epoch budget selectively: favor families or runs that actually hit the budget ceiling
    5. Seed diversity: Use top-N from multiple metrics (Obj, S2, S3, PnL, Sharpe) to avoid converging on one metric that doesn't predict real performance
