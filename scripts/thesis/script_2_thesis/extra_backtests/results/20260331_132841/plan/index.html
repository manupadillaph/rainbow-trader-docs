    Script 2 Extra Backtests Plan

    What This File Is

    What this section is: an automatically generated shortlist of friction-victim Script 2 trials selected from the database, plus proposed higher entry-threshold reruns.

    • Database: database/rainbow_trader.sqlite
    • Trials scanned: 1837
    • Candidates selected: 20
    • Target per fold: 5
    • Minimum trades filter: 100
    • Output folder: scripts/thesis/script_2_thesis/extra_backtests/results/20260331_132841

    Selection Rules

    • Completed Script 2 trainable trials only.
    • Negative realized PnL required.
    • Positive PnL + Est.Fric required.
    • Minimum trade count required.
    • Candidates are ranked inside each fold separately, then the top per-fold-k are kept from each fold.
    • Final score emphasizes Objective, S2, Entry F1, F1 Macro, low JS, gross-edge proxy, and friction drag.
    • Threshold proposals come from the original prediction parquets and are meant for rerun selection, not for guaranteed trade-count targeting.

    Column Legend

    • Thr Orig: original entry_threshold stored in the experiment strategy config for the baseline trial.
    • Thr Rec: recommended stricter entry_threshold inferred from the trial's prediction parquet.
    • Thr Alt: alternate stricter threshold, usually more aggressive than Thr Rec.
    • Keep Rec: share of rows that pass the current confidence gate and would still pass after switching to Thr Rec.
    • Keep Alt: same retention concept, but for Thr Alt.
    • BT Prog%: completion ratio of the original stage-3 backtest, computed as processed_count / prediction_parquet_rows.
