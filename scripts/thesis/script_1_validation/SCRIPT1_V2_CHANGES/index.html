    Script 1 — V2 Changes Log

    This document tracks the evolution from V1 → V2 of Script 1 experiment configurations.

    • V1 run: 20260306_204834
    • V1 findings: results/20260306_204834/script1_findings_and_recommendations.md
    • V2 changes applied: 2026-03-07

    1. V1 Summary — What Was Run

    V1 Configuration (shared across all 8 experiments)

    n_trials:                          12
learning_rate:                     fixed 0.0001
batch_size:                        fixed 128
weight_decay:                      fixed 0.0001
window_size:                       int range [20, 48]  ← ONLY tunable parameter
epochs:                            15
early_stopping_patience:           5

pruning_config:
  degenerate:                      warmup_epochs=3, patience_epochs=5
  untradeable:                     warmup_epochs=3, patience_epochs=5
  distribution_drift:              warmup_epochs=3, patience_epochs=5

Leak feature (E04, E07):          pipeline=[]  (NO normalization)

    V1 Results

    Exp Description entry_f1 median Status
    S1-E01 Dummy + ViT 0.994 ✅ PASS
    S1-E02 Dummy + Random 0.262 ✅ Reference
    S1-E03 Random + ViT 0.000 ⚠️ All-NONE 100%
    S1-E04 Random + ViT + Leak 0.000 ⚠️ All-NONE 64%
    S1-E05 Random + Label Copy 1.000 ✅ PASS
    S1-E06 Random + Random 0.251 ✅ Reference
    S1-E07 Dummy + ViT + Leak 0.000 ⚠️ All-NONE 83%
    S1-E08 Dummy + Label Copy 1.000 ✅ PASS

    V1 Root Causes (see findings doc for full evidence)

    1. RC-1 (CRITICAL): Leak feature has pipeline: [] — raw future close prices dominate normalized features
    2. RC-2 (CRITICAL): All hyperparameters fixed except window_size — zero training diversity
    3. RC-3 (HIGH): Early stopping (patience=5, epoch ~5-6) fires BEFORE internal pruning (warmup=3 + patience=5 = epoch 8) can detect degenerate behavior
    4. RC-4 (MEDIUM): Class imbalance (NONE ~77%) creates strong all-NONE attractor with fixed LR

    2. V2 Changes — What We're Fixing

    Change 1 — CRITICAL: Normalize leak feature pipeline (E04, E07 only)

    Before (V1):

    {
  "feature_type": "INDICATOR",
  "indicator": { "indicator_type": "LEAK", ... },
  "columns": [{ "selected": "leak" }],
  "pipeline": []
}

    After (V2):

    {
  "feature_type": "INDICATOR",
  "indicator": { "indicator_type": "LEAK", ... },
  "columns": [{ "selected": "leak" }],
  "pipeline": [
    { "pipeline_type": "NORMALIZATION" },
    { "pipeline_type": "RELATIVE_INDICATOR" }
  ]
}

    Why: The leak indicator is an indicator feature, so it must use indicator pipelines (RELATIVE_INDICATOR and/or NORMALIZATION) rather than price pipelines. This keeps transformations aligned with per-indicator metadata (relative_meta, normalization_meta) and avoids invalid OHLCV requirements at fetch/process stage.

    Affects: S1-E04, S1-E07

    Change 2 — HIGH: Add learning_rate as tunable (ViT experiments only)

    Before (V1):

    "learning_rate": { "type": "fixed", "value": 0.0001 }

    After (V2):

    "learning_rate": { "type": "categorical", "choices": [3e-5, 5e-5, 1e-4, 2e-4, 3e-4, 5e-4] }

    Why: With fixed LR=0.0001 on data with strong class imbalance, the model consistently collapses to all-NONE. Higher LRs (3e-4, 5e-4) produce more aggressive gradient updates that can escape the all-NONE local minimum. Lower LRs (3e-5) provide fine-grained tuning for easier signals. Optuna can now explore which LR works best per data type.

    Affects: S1-E01, S1-E03, S1-E04, S1-E05, S1-E07, S1-E08 (all ViT experiments)

    Does NOT affect: S1-E02, S1-E06 (random models don't train with gradient descent)

    Change 3 — HIGH: Fix early stopping vs internal pruning race (all experiments)

    Before (V1):

    "degenerate":        { "warmup_epochs": 3, "patience_epochs": 5 }
"untradeable":       { "warmup_epochs": 3, "patience_epochs": 5 }
"distribution_drift": { "warmup_epochs": 3, "patience_epochs": 5 }

    Internal pruning fires at: warmup(3) + patience(5) = epoch 8 Early stopping fires at: patience(5) = epoch ~5-6 Result: Internal pruning NEVER acts.

    After (V2):

    "degenerate":        { "warmup_epochs": 2, "patience_epochs": 2 }
"untradeable":       { "warmup_epochs": 2, "patience_epochs": 2 }
"distribution_drift": { "warmup_epochs": 2, "patience_epochs": 2 }

    Internal pruning fires at: warmup(2) + patience(2) = epoch 4 Early stopping fires at: patience(5) = epoch ~5-6 Result: Internal pruning fires BEFORE early stopping. Degenerate trials get pruned, Optuna learns to avoid those hyperparameter regions.

    Also updating defaults to match:

    "default_warmup_epochs": 2, "default_patience_epochs": 2

    Affects: All 8 experiments

    Change 4 — MEDIUM: Increase n_trials from 12 to 20 (all experiments)

    Before (V1): "n_trials": 12

    After (V2): "n_trials": 20

    Why: With LR now having 6 categorical values and window_size ranging 20-48, 12 trials is insufficient for Optuna to explore the space. 20 trials gives better coverage while keeping run time reasonable.

    Affects: All 8 experiments

    Change 5 — LOW: Add batch_size as tunable (ViT experiments only)

    Before (V1):

    "batch_size": { "type": "fixed", "value": 128 }

    After (V2):

    "batch_size": { "type": "categorical", "choices": [64, 128, 256] }

    Why: Different batch sizes change optimization dynamics. Smaller batches (64) provide noisier gradients that can help escape local minima. Larger batches (256) provide smoother gradients for stable convergence. This adds another dimension for Optuna to explore.

    Affects: S1-E01, S1-E03, S1-E04, S1-E05, S1-E07, S1-E08 (all ViT experiments)

    Does NOT affect: S1-E02, S1-E06 (random models)

    Change 6 — Switch to mini backtest for Script 1 (added 2026-03-09)

    Before:

    "run_full_backtest_on_trial": true

    After:

    "run_full_backtest_on_trial": false
"last_timestamps_count": 100    (in sampling_config)

    Why: Script 1 pass/fail criteria only use stage2 metrics (entry_f1). Stage3 backtest data (return_pct, trades_count) is collected but never used in any test. Running full backtest on every trial wastes significant time. Mini backtest with 100 tail rows is sufficient for stage3 scoring contribution while being extremely fast.

    Affects: ALL Script 1 experiments (E01–E08)
