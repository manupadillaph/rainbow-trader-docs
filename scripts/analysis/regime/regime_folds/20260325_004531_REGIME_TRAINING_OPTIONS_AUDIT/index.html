    Regime Training Options Audit

    Scope

    This audit compares three regime views built from the same BTCUSDT history:

    • 5m -> daily dominant from 20260324_154928_BTCUSDT_5m_regime_history
    • 1d native from 20260324_162703_BTCUSDT_1d_native_regime_history
    • 1w native from 20260324_162703_BTCUSDT_1w_native_regime_history

    The goal is not to describe artifacts again. The goal is to decide how to replace arbitrary 3-month Script2 folds with training logic that is more regime-aware and more operationally meaningful.

    Main Findings

    1. 5m -> daily dominant is useful for diagnosis, not for primary fold construction

    Evidence:

    • taxonomy_27_full top state TRENDING | HIGH_VOL | BEARISH covers 20.45% of days, but its average period length is only 1.4308 days and its maximum is 8.0 days.
    • taxonomy_9_trend_volatility top state RANGING | LOW_VOL covers 30.35% of days, but average period length is only 1.6047 days and maximum is 8.0 days.
    • taxonomy_9_trend_direction top state TRENDING | BEARISH covers 39.53% of days, but average period length is only 1.6567 days and maximum is 7.0 days.
    • Every daily threshold summary for 30d, 60d, and 90d returns zero.

    Interpretation:

    • This view is still too twitchy to define contiguous training folds.
    • Even after collapsing intraday states into daily winners, the state still flips too often.
    • This does not mean the view is useless. It is very valuable for label diagnostics and directional skew.

    Useful evidence from labels:

    • taxonomy_9_trend_direction shows strong directional separation:
    • TRENDING | BULLISH: 15.25% tradable share, 43.89 tradable labels/day, 34.52% directional imbalance.
    • TRENDING | BEARISH: 15.08% tradable share, 43.40 tradable labels/day, -35.41% directional imbalance.
    • taxonomy_9_trend_volatility shows volatility matters for opportunity density:
    • TRENDING | HIGH_VOL: 17.16% tradable share, 49.43 tradable labels/day.
    • RANGING | LOW_VOL: 12.71% tradable share, 36.60 tradable labels/day.

    Conclusion:

    • Keep this view as a diagnostic overlay.
    • Do not use it as the primary fold engine.

    2. 1d native is the best current backbone for regime-oriented training

    Evidence:

    • taxonomy_27_full:
    • TRENDING | NORMAL_VOL | BULLISH: 23.58% of bars, 21 periods, average 19.95 days, max 62.0 days.
    • TRENDING | NORMAL_VOL | BEARISH: 24.42%, 35 periods, average 12.4 days, max 60.0 days.
    • taxonomy_9_trend_volatility:
    • TRENDING | NORMAL_VOL: 53.57%, 46 periods, average 20.70 days, max 64.0 days.
    • RANGING | NORMAL_VOL: 22.73%, average 12.63 days, max 39.0 days.
    • taxonomy_9_trend_direction:
    • TRENDING | BULLISH: max 69.0 days.
    • TRENDING | BEARISH: max 69.0 days.
    • Threshold summary is finally operational:
    • taxonomy_27_full: 30 periods with at least 12 bars.
    • taxonomy_9_trend_volatility: 46 periods with at least 12 bars.
    • taxonomy_9_trend_direction: 27 periods with at least 12 bars.

    Interpretation:

    • This is the first view that produces blocks long enough to become real train/validation units.
    • It is much more stable than 5m -> daily dominant, but still detailed enough to separate bullish versus bearish trend environments.
    • It looks like the best current compromise between stability and information content.

    Useful evidence from labels:

    • Label density is much flatter than in 5m -> daily dominant.
    • That is not a weakness. It means the macro state is smoother and less biased by intraday noise.
    • taxonomy_9_trend_volatility:
    • TRENDING | NORMAL_VOL: 14.49% tradable share.
    • RANGING | NORMAL_VOL: 14.08%.
    • NEUTRAL | NORMAL_VOL: 14.81%.
    • taxonomy_9_trend_direction:
    • TRENDING | BULLISH: 14.92% tradable share.
    • TRENDING | BEARISH: 13.77%.

    Conclusion:

    • This is the best candidate for replacing arbitrary 3-month folds.
    • If we redesign training now, this should be the main regime axis.

    3. 1w native is excellent as a macro filter, but too coarse to be the only training axis

    Evidence:

    • taxonomy_27_full:
    • TRENDING | NORMAL_VOL | BULLISH: average 75.83 days, max 224.0 days.
    • TRENDING | NORMAL_VOL | BEARISH: average 51.8 days, max 147.0 days.
    • taxonomy_9_trend_volatility:
    • TRENDING | NORMAL_VOL: 57.08%, average 65.15 days, max 259.0 days.
    • taxonomy_9_trend_direction:
    • TRENDING | BULLISH: max 224.0 days.
    • TRENDING | BEARISH: average 100.33 days, max 210.0 days.
    • taxonomy_3_volatility is almost collapsed:
    • NORMAL_VOL dominates 93.40% of bars with max 553.0 days.

    Interpretation:

    • This view gives very strong persistence.
    • It is almost too stable.
    • Several taxonomies compress market history into a few giant states, which is good for macro context but weak for fine specialization.

    Useful evidence from labels:

    • Many label distributions become very flat:
    • taxonomy_3_trend: TRENDING 14.72%, NEUTRAL 14.78%, RANGING 14.98%.
    • taxonomy_9_trend_volatility: most large groups cluster around 14.76% to 14.84%.
    • That suggests this view is better for regime routing or top-level gating than for per-trade specialization.

    Conclusion:

    • Use 1w native as a macro regime filter or router.
    • Do not make it the only fold definition.

    What The Results Suggest About Training

    What should stop

    • Stop treating arbitrary 3-month windows as the default scientific unit.
    • Stop assuming a temporal fold is meaningful just because it is contiguous in time.
    • Stop trying to force 27 exact daily-dominant states from 5m into contiguous folds. The evidence says they are too unstable.

    What is already justified now

    • It is already justified to redesign experiments around 1d native regimes.
    • It is not necessary to fully finish the persistence state machine before starting the first regime-oriented training redesign.
    • The current detector is imperfect, but it is already good enough to support a first generation of regime-based training experiments at the 1d level.

    What is not justified yet

    • It is not justified to jump directly to 27 regime-specialized models.
    • It is not justified to use only 1w native regimes as the full training axis.
    • It is not justified to assume transitions are harmless. We still need explicit transition handling before any live-routing logic.

    Real Training Options

    Option A. 1d native pooled same-regime training

    How it works:

    • Pick a 1d native taxonomy, preferably taxonomy_9_trend_direction or taxonomy_9_trend_volatility.
    • Collect all segments from the same regime family across multiple years.
    • Train on pooled non-contiguous history from that regime family.
    • Validate on later held-out segments from the same regime family.

    Why it is attractive:

    • It directly answers your thesis: train on same market type, not arbitrary calendar windows.
    • It gives more data than one contiguous block.
    • It matches the evidence that 1d native states are stable enough to define meaningful segments.

    Recommended first version:

    • Start with taxonomy_9_trend_direction.
    • Families:
    • TRENDING | BULLISH
    • TRENDING | BEARISH
    • RANGING | BULLISH
    • RANGING | BEARISH
    • Skip tiny families until we have persistence logic.

    Option B. 1d native contiguous-block training

    How it works:

    • Use the longest contiguous 1d native periods as train/validation units.
    • Example: train on one or more long TRENDING | NORMAL_VOL | BULLISH blocks, validate on a later bullish block of the same family.

    Why it is attractive:

    • Cleaner scientifically.
    • Easier to explain than pooled non-contiguous training.
    • Good first benchmark against current Script2 folds.

    Why it is weaker than Option A:

    • Less data.
    • More variance from one block to another.
    • Risk of over-reading one specific bull or bear episode.

    Option C. Two-stage hierarchy: 1w native router, 1d native specialist

    How it works:

    • First detect the macro regime with 1w native.
    • Inside that macro regime, select a 1d native specialist model or specialist hyperparameter family.

    Example:

    • If 1w native says TRENDING | NORMAL_VOL | BULLISH, route to the 1d native bullish-trend family.
    • If 1w native says TRENDING | NORMAL_VOL | BEARISH, route to bearish specialists.

    Why it is attractive:

    • Uses the best property of each detector:
    • 1w native: strong persistence
    • 1d native: useful specialization
    • This looks like the most realistic path toward a live model router.

    Why not first:

    • It is more complex operationally.
    • Better as phase 2 after proving that 1d native specialization actually helps.

    Option D. Global model plus regime features

    How it works:

    • Keep one global training set.
    • Add regime context as features:
    • 1d native taxonomy
    • 1w native taxonomy
    • maybe later confirmed versus candidate regime

    Why it is attractive:

    • Lowest operational complexity.
    • Easy baseline.
    • Lets us test whether regime information helps before committing to multiple models.

    Why it may underperform:

    • The model still has to learn very different behaviors in one parameter space.
    • Strong regime-specific edges can get blurred.

    Option E. Regime-aware do-not-trade policy

    How it works:

    • Use regime only to block trades in low-information or noisy environments.
    • Candidate sources:
    • 5m -> daily dominant low dominance days
    • future MIXED / UNSTABLE states
    • low tradable-density groups

    Why it matters:

    • One likely outcome of this research is not only “which model to use”, but also “when not to trade”.
    • Some environments may be better handled by no position than by a weaker specialist.

    What I Would Actually Do Next

    Phase 1: first real redesign

    I would start now with:

    1. 1d native as the main regime backbone.
    2. taxonomy_9_trend_direction as the first family to test.
    3. Pooled same-regime training across non-contiguous segments.
    4. Held-out later same-regime validation.
    5. A plain baseline with one global model for comparison.

    This is the cleanest first experiment that actually attacks the thesis.

    Phase 2: macro conditioning

    If phase 1 shows signal:

    1. add 1w native as a macro router or gating feature
    2. compare:
    3. global model
    4. 1d native specialist models
    5. 1w -> 1d hierarchical routing

    Phase 3: transition safety

    After that:

    1. implement persistence
    2. implement candidate versus confirmed
    3. add MIXED / UNSTABLE
    4. test whether transition periods are where models break

    My Recommendation On The Detector Plan

    I would not pause everything to perfect the detector first.

    I would do this:

    • Proceed now with regime-oriented training design using current 1d native output.
    • Continue the follow-up plan selectively, not sequentially.

    Priority order:

    1. Build the first 1d native regime-based training experiments.
    2. Map Script2 folds and model outcomes onto 1d native and 1w native regimes.
    3. Then add persistence and transition logic.
    4. Only after that consider HMM, Markov switching, or change-point benchmarks.

    Why:

    • The current evidence is already strong enough to reject arbitrary 3-month folds as the main research design.
    • The current evidence is already strong enough to justify 1d native regime-oriented experiments.
    • The next detector improvements are important, but they should refine the regime framework, not block it.

    Bottom Line

    • 5m -> daily dominant: keep as diagnostic overlay.
    • 1d native: use as the main training regime axis now.
    • 1w native: use as macro context or routing layer, not as the only fold definition.
    • Best first experiment: pooled same-regime training on 1d native, with held-out later same-regime validation and a global-model baseline beside it.
