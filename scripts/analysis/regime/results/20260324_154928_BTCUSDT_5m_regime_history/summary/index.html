    BTC Regime + Script2 Label Analysis

    What This File Is

    What this section is: a short entry point for the run, with pointers to the more focused summaries.

    • Daily regime file: summary_daily_regimes.md
    • Label distribution file: summary_label_distributions.md
    • Intraday fragmentation file: summary_intraday_fragmentation.md

    All Generated Artifacts

    What this section is: a full inventory of what this script writes into the run folder, explained in simple language so each file has a clear purpose.

    • run_context.json: the machine-readable receipt for the run. It records the exact input range, parquet ids, row counts, and Script2 labeling settings used.
    • summary.md: the front door. Read this first to understand what the run is, what files exist, and where to go next.
    • summary_daily_regimes.md: the daily regime notebook. It shows which basket owned each day and how long those daily stretches lasted.
    • summary_label_distributions.md: the labels notebook. It shows how LONG, SHORT, and NONE labels distribute inside each daily regime basket.
    • summary_intraday_fragmentation.md: the fragmentation notebook. It explains how noisy the exact 5m regime states are before we collapse them by day.
    • daily_regimes.csv: one row per day. This is the daily market map with the winning basket for each 3/9/27 taxonomy and the dominance percentage of that winner.
    • daily_regimes_with_labels.csv: one row per day plus daily label totals. This is the easiest place to spot days with many tradable labels or days that are mostly NONE.
    • labels_with_daily_regimes.parquet: one row per label timestamp joined with its day's dominant regime baskets. Use this when you want row-level analysis instead of daily aggregates.
    • daily_taxonomy_*_coverage.csv: one file per taxonomy, grouped by daily dominant basket. These files answer 'how many days did each basket own, and how persistent was it across daily periods?'.
    • daily_taxonomy_*_periods.csv: one file per taxonomy, one row per contiguous daily block. These files answer 'when did this basket start, when did it end, and how long did it last?'.
    • intraday_taxonomy_*_coverage.csv: one file per taxonomy, grouped from exact 5m states. These files answer 'which exact states consume the most bars even if they do not stay stable for long?'.
    • intraday_taxonomy_*_periods.csv: one file per taxonomy, one row per uninterrupted 5m exact-state run. These files are for diagnosing twitchiness, not for proposing folds directly.
    • label_distribution_taxonomy_*.csv: one file per taxonomy showing label behavior by daily dominant basket. These files answer 'where are the LONGs, where are the SHORTs, and where is trading mostly absent?'.

    Interpretation Hints

    What this section is: reading guidance so the daily, label, and intraday files are not confused with each other.

    • Intraday exact-state periods remain useful diagnostically, but they are expected to fragment heavily.
    • Daily dominant baskets are the fold-oriented view: they tell us what regime owned each day.
    • Label distributions over daily dominant baskets help reveal long/short dominance and sparse-trade days.
    • If a daily regime has many days but very few tradable labels, it may be a natural do not trade candidate.
