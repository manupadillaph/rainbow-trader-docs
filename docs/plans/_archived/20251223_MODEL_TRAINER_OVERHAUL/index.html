    Model Trainer & Small Window Preparer Overhaul

    Status: 🟢 COMPLETE
    Started: 2025-12-22
    Completed: 2025-12-23
    Priority: HIGH

    📋 Description

    Complete overhaul of the model-trainer and window-preparer modules. Both modules currently contain mockup code with time.sleep() placeholders. This plan covers:

    1. Moving existing code to _OLD/ directories
    2. Rebuilding both modules from scratch using fat-window-builder as the structural template
    3. Implementing Optuna integration for hyperparameter optimization, feature selection, and column weight optimization
    4. Implementing Small Window preparation with hash-based caching
    5. Updating database schema (trials table) and configuration types
    6. Adding cleanup command to housekeeping module

    🧩 Rules

    Global rules (do not remove)

    • README template: All module READMEs must follow docs/README_MODULE_TEMPLATE.md.
    • Read from disk before each iteration: Before starting work, re-open this plan, the relevant docs, and the code you will touch. Reality lives on disk (plans, docs, code), not in cached summaries.
    • Git workflow: Use the MCP GitHub tools and follow docs/GIT-COMMITS.md. Never run git add from the repository root unless explicitly specified in this plan.
    • Respect project docs: All changes must comply with the relevant docs (configuration, code generation, database schema, markdown style, etc.).
    • Language: User responses and AI interpretations in the conversation log may be in Spanish, but the work plan document itself and all generated code/docs MUST be written in English.

    Plan-specific rules

    • Template module: Use fat-window-builder as the structural template for both modules. Copy patterns for: service class, CLI, types, helpers, constants.
    • No legacy code: After moving to _OLD/, the new implementation starts from scratch. Do not copy logic from old files, only structure patterns.
    • Optuna version: Use Optuna latest stable (already in requirements). Sampler: TPESampler. Pruner: MedianPruner.
    • Sync execution: The model-trainer runs Optuna's study.optimize() synchronously. Window-preparer is called as a function, NOT as a separate process with events.
    • Cache by hash: Small Windows use hash-based caching. Same config → same parquet → reuse.
    • Column-level selection: Optuna selects at column level (not feature level). Each column has weight_range and mandatory fields.
    • Weights in features: Weight configuration lives in ColumnOutputInfo (inside FatWindow features), NOT in optuna_config.

        """

        2. Extract all columns from FAT window config with their weight_range/mandatory
        3. Create Optuna study with TPESampler + MedianPruner
        4. Define objective function that:
           a. Suggests window_size from optuna_config.window_size
           b. Suggests weight per column within its weight_range
           c. Calls SmallWindowPreparer to get/create small window
           d. Trains model with suggested hyperparameters
           e. Returns validation metric
        5. Run study.optimize(objective, n_trials=optuna_config.n_trials)
        6. Save all trials to DB
        7. Emit ALL_TRIALS_DONE event if emit=True
        8. Return best trial info
        """
    • Objective function structure:
    def _create_objective(
    self,
    experiment_config: ExperimentConfig,
    fat_parquet: Parquet,
    all_columns: list[ColumnOutputInfo],
) -> Callable[[optuna.Trial], float]:
    """Create the objective function for Optuna."""

    def objective(trial: optuna.Trial) -> float:
        # 1. Suggest window size
        ws_param = experiment_config.optuna_config.window_size
        window_size = suggest_param(trial=trial, name="window_size", param=ws_param)

        # 2. Suggest column weights
        selected_columns: dict[str, float] = {}
        for col in all_columns:
            col_name = col.output_name or col.selected
            min_w, max_w = col.weight_range
            weight = trial.suggest_float(f"w_{col_name}", min_w, max_w)
            if weight > 0:
                selected_columns[col_name] = weight

        # 3. Validate minimum features selected
        min_features = experiment_config.optuna_config.feature_selection.min_features
        if len(selected_columns) < min_features:
            # Optuna handles this gracefully - bad trial
            return 0.0

        # 4. Prepare small window
        small_result = self.small_window_preparer.prepare_small_window(
            experiment_id=experiment_config.experiment_id,
            trial_id=trial.number,
            fat_parquet_id=fat_parquet.id,
            window_size=window_size,
            selected_columns=selected_columns,
        )

        # 5. Suggest model hyperparameters
        model_params = self._suggest_model_params(trial, experiment_config)

        # 6. Train model with pruning callbacks
        val_metric = self._train_and_evaluate(
            small_parquet=small_result.parquet,
            model_params=model_params,
            trial=trial,  # For intermediate reporting
        )

        # 7. Save trial to DB
        self._save_trial_to_db(trial, selected_columns, model_params, val_metric)

        return val_metric

    return objective

) -> float:

    # Create model
