    Event Taxonomy — Simple View

    What this is: A child-friendly view of the event refactor. Source of truth: EVENT_TAXONOMY_PLAN.md Generated: 2026-03-24

    1. Final Events (what will exist after the refactor)

    Every module job follows the same pattern: REQUESTED → STARTED → PROGRESS → DONE.

    Data Fetch

    Event Who emits it
    DATA_FETCH_REQUESTED Orchestrator
    DATA_FETCH_STARTED Data-Fetcher
    DATA_FETCH_PROGRESS Data-Fetcher
    DATA_FETCH_DONE Data-Fetcher

    Labels

    Event Who emits it
    LABELS_REQUESTED Orchestrator
    LABELS_STARTED Labeling-Service
    LABELS_PROGRESS Labeling-Service
    LABELS_DONE Labeling-Service

    FAT Windows

    Event Who emits it
    FAT_WINDOWS_REQUESTED Orchestrator
    FAT_WINDOWS_STARTED Window-Preparer
    FAT_WINDOWS_PROGRESS Window-Preparer
    FAT_WINDOWS_DONE Window-Preparer

    SMALL Windows

    Event Who emits it
    SMALL_WINDOWS_REQUESTED Orchestrator
    SMALL_WINDOWS_STARTED Window-Preparer
    SMALL_WINDOWS_PROGRESS Window-Preparer
    SMALL_WINDOWS_DONE Window-Preparer

    Data Processing

    Event Who emits it
    DATA_PROCESSING_REQUESTED Orchestrator
    DATA_PROCESSING_STARTED Data-Processing
    DATA_PROCESSING_PROGRESS Data-Processing
    DATA_PROCESSING_DONE Data-Processing

    Training (container — contains Phases, which contain Trials)

    Event Who emits it Notes
    TRAINING_REQUESTED Orchestrator
    TRAINING_STARTED Model-Trainer
    TRAINING_PROGRESS Model-Trainer Cascaded up from every epoch + every trial end + every phase end
    TRAINING_DONE Model-Trainer NEW — emitted when all phases finish

    Phase (nested inside Training)

    Event Who emits it Notes
    PHASE_STARTED Model-Trainer NEW
    PHASE_PROGRESS Model-Trainer NEW — cascaded up from every epoch + every trial end
    PHASE_DONE Model-Trainer NEW — replaces old ALL_TRIALS_DONE

    Trial (nested inside Phase)

    Event Who emits it Notes
    TRIAL_STARTED Model-Trainer Already exists
    TRIAL_PROGRESS Model-Trainer NEW — emitted every epoch
    TRIAL_DONE Model-Trainer NEW — replaces old TRIAL_COMPLETED + TRIAL_PRUNED + TRIAL_FAILED (outcome in payload)

    Backtest

    Event Who emits it
    BACKTEST_REQUESTED Orchestrator
    BACKTEST_STARTED Backtesting-Engine
    BACKTEST_PROGRESS Backtesting-Engine
    BACKTEST_DONE Backtesting-Engine

    Live Trading

    Event Who emits it Notes
    LIVE_TRADING_REQUESTED Orchestrator NEW
    LIVE_TRADING_STARTED Live-Trading-Runner Already exists
    LIVE_TRADING_PROGRESS Live-Trading-Runner Replaces old LIVE_EXEC_SUMMARY
    LIVE_TRADING_DONE Live-Trading-Runner Replaces old LIVE_TRADING_STOPPED
    LIVE_TRADING_ANOMALY_DETECTED Live-Trading-Runner Replaces old LIVE_ANOMALY_DETECTED

    HITL (Human-in-the-Loop)

    Event Who emits it
    HITL_REQUEST Orchestrator
    HITL_DECISION Control-Panel
    HITL_APPROVED Control-Panel
    HITL_REJECTED Control-Panel

    Total canonical events: 45

    2. Renames (old name → new name)

    Old name (in code today) New name Why
    ALL_TRIALS_DONE PHASE_DONE It was actually end-of-phase, not end-of-training
    TRIAL_COMPLETED TRIAL_DONE Unified terminal event with outcome in payload
    TRIAL_PRUNED TRIAL_DONE Same — outcome="pruned" in payload
    TRIAL_FAILED TRIAL_DONE Same — outcome="failed" in payload
    BACKTEST_TRIAL_DONE BACKTEST_DONE Follow {OPERATION}_DONE convention
    DATA_PROCESSED_DONE DATA_PROCESSING_DONE Follow {OPERATION}_DONE convention
    LIVE_EXEC_SUMMARY LIVE_TRADING_PROGRESS Was the heartbeat — now follows naming convention
    LIVE_TRADING_STOPPED LIVE_TRADING_DONE Terminal event must be _DONE
    LIVE_ANOMALY_DETECTED LIVE_TRADING_ANOMALY_DETECTED Must share the LIVE_TRADING_ prefix

    Renames in docs only (these names exist in EVENTS.md/MASTER-PLAN.md but not in code)

    Doc-only name Correct name
    BACKTEST_COMPLETE BACKTEST_DONE
    DATA_FETCH_COMPLETE DATA_FETCH_DONE
    LABELING_COMPLETE LABELS_DONE
    TRAINING_COMPLETE TRAINING_DONE
    TRIAL_SUMMARY_READY removed (was never real)

    3. Deletions from code (events in EventType enum today that will be removed)

    Event to delete Why
    TRIAL_TRAINING_DONE Dead code — never emitted. Covered by TRIAL_DONE
    TRIAL_COMPLETED Replaced by TRIAL_DONE (outcome="completed")
    TRIAL_PRUNED Replaced by TRIAL_DONE (outcome="pruned")
    TRIAL_FAILED Replaced by TRIAL_DONE (outcome="failed")
    TRIAL_CANCELLED Folded into TRIAL_DONE (outcome="cancelled")
    TRIAL_STOPPING No real stop flow exists — removed
    ALL_TRIALS_DONE Renamed to PHASE_DONE
    BACKTEST_TRIAL_DONE Renamed to BACKTEST_DONE
    DATA_PROCESSED Legacy/deprecated — replaced by DATA_PROCESSING_DONE
    DATA_PROCESSED_DONE Renamed to DATA_PROCESSING_DONE
    DATA_PIPELINE_REQUESTED Dead code — never emitted, redundant concept
    DATA_PIPELINE_DONE Dead code — never emitted, redundant concept
    LIVE_EXEC_SUMMARY Renamed to LIVE_TRADING_PROGRESS
    LIVE_TRADING_STOPPED Renamed to LIVE_TRADING_DONE
    LIVE_ANOMALY_DETECTED Renamed to LIVE_TRADING_ANOMALY_DETECTED
    TRIAL_SCORED No runtime emitter — ranking belongs in DB, not events
    TRIAL_CONFIG_READY No requester/consumer contract
    ANALYST_TRIAL_SUMMARY_READY No emitter callsite — future artifact flow
    ANALYST_EXPERIMENT_SUMMARY_READY No emitter callsite — future artifact flow
    EXPERIMENT_STOPPING Too vague — stop flows must be domain-specific
    EXPERIMENT_STOPPED Too vague — same reason
    EXECUTION_PLAN_CREATED Artifact event — no lifecycle contract
    EXPERIMENT_PLAN_CREATED Artifact event — no lifecycle contract
    MASTER_EXEC_PLAN_UPDATED No emitter — no lifecycle contract
    HOUSEKEEPING_DONE No REQUESTED/STARTED/PROGRESS flow — incomplete lifecycle
