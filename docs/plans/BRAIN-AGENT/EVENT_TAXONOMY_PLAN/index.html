    Event Taxonomy Refactor

    Status: 🔵 STAGE 3 — Task Planning Started: 2026-03-19 Priority: HIGH

    📋 Description

    Refactor the event system to establish a coherent, consistent, and logically correct taxonomy across all modules. The current event enum contains broken naming patterns, redundant events, and missing events. This plan defines the approved taxonomy and derives all implementation tasks from it.

    Predecessor audit: ./BRAIN_FOUNDATION_AUDIT_PLAN.md (Stage 1 completed)

    Approved taxonomy (user-confirmed 2026-03-19):

    Training (container)
  └── Phase (optimization phase within a training run)
        └── Trial (single Optuna iteration within a phase)
              └── Epoch (training loop iteration within a trial)

    Cascade rule: Every epoch emits events at all 3 levels. Every trial-end emits at 2 levels. Every phase-end emits at 1 level.

    Trigger Events emitted
    Epoch completes TRIAL_PROGRESS + PHASE_PROGRESS + TRAINING_PROGRESS
    Trial completes TRIAL_DONE + PHASE_PROGRESS + TRAINING_PROGRESS
    Phase completes PHASE_DONE + TRAINING_PROGRESS

    Normalization decisions locked on 2026-03-20:

    • Batch workflows: Use _REQUESTED → _STARTED → _PROGRESS → _DONE.
    • Live trading: It follows the same naming convention as the other modules. Its normalized family is LIVE_TRADING_REQUESTED → LIVE_TRADING_STARTED → LIVE_TRADING_PROGRESS → LIVE_TRADING_DONE, plus LIVE_TRADING_ANOMALY_DETECTED for safety incidents.
    • Heartbeat rename: LIVE_EXEC_SUMMARY is renamed to LIVE_TRADING_PROGRESS.
    • Inventory source: The keep/rename/delete/new decisions in this plan are lowered from BRAIN_FOUNDATION_AUDIT_PLAN.md section F16.B.

    Plan-specific rules

    • Enum is source of truth: modules/common/src/rainbow/common/types/events.py is the single source of truth for EventType. All docs derive from it.
    • No compat aliases: Do not create compatibility aliases for deleted event names. Rename directly — all callsites must be updated in the same PR.
    • Payload catalog: The target payload catalog for the final canonical set is part of this plan and becomes the contract implementation must follow.
    • Payload schemas required: Every emitted event must have a Pydantic payload model in EVENT_PAYLOAD_MAP. Raw dict payloads are not acceptable for emitted events.

    • Tests must reflect new names: Any test that references a deleted or renamed EventType must be updated.

    Finding 1: Training→Phase→Trial hierarchy missing from event system

    • Problem: The event system has no PHASE_* events and no TRAINING_DONE event. The existing ALL_TRIALS_DONE event actually represents end-of-phase (it has phase_number in payload), not end-of-training. The hierarchy Training→Phase→Trial is not reflected in the event taxonomy.
    • Location:
    • modules/common/src/rainbow/common/types/events.py — EventType enum, missing PHASE_STARTED/PROGRESS/DONE, TRAINING_DONE
    • modules/model-trainer/src/rainbow/model_trainer/helpers/optuna/trials.py:945 — emits ALL_TRIALS_DONE with phase_number in payload
    • Impact: The orchestrator cannot distinguish end-of-phase from end-of-training. No event signals that run_optimization() completed all phases.
    • Recommendation: Add PHASE_STARTED, PHASE_PROGRESS, PHASE_DONE (replacing ALL_TRIALS_DONE). Add TRAINING_DONE. Keep TRAINING_STARTED (already declared, not emitted).
    • User Decision: Approved 2026-03-19. Taxonomy confirmed: Training→Phase→Trial with cascade progress events.
    • Action Required: Add PHASE_STARTED, PHASE_PROGRESS, PHASE_DONE to enum. Rename ALL_TRIALS_DONE → PHASE_DONE. Implement emission in model-trainer.

    Finding 2: TRIAL_COMPLETED/PRUNED/FAILED must consolidate into TRIAL_DONE

    • Problem: Three separate terminal events for a trial (TRIAL_COMPLETED, TRIAL_PRUNED, TRIAL_FAILED) break the *_DONE convention. The outcome (completed/pruned/failed/cancelled) belongs in the payload, not in the event name.
    • Location:
    • modules/common/src/rainbow/common/types/events.py — three separate EventType values
    • modules/model-trainer/src/rainbow/model_trainer/helpers/optuna/objective.py:635 — emits TRIAL_COMPLETED
    • modules/model-trainer/src/rainbow/model_trainer/helpers/optuna/objective.py:696 — emits TRIAL_PRUNED
    • modules/model-trainer/src/rainbow/model_trainer/helpers/optuna/objective.py:816 — emits TRIAL_FAILED
    • Impact: Inconsistent with the *_DONE convention. Orchestrator must listen to 3 events instead of 1.
    • Recommendation: Delete TRIAL_COMPLETED, TRIAL_PRUNED, TRIAL_FAILED. Add TRIAL_DONE with payload outcome: Literal["completed", "pruned", "failed", "cancelled"].
    • User Decision: Approved 2026-03-19. Single TRIAL_DONE with outcome in payload.
    • Action Required: Replace 3 enum values with TRIAL_DONE. Update all 3 emit callsites in objective.py. Add Pydantic schema TrialDonePayload.

    Finding 3: TRIAL_PROGRESS missing (cascade rule)

    • Problem: TRIAL_PROGRESS is not declared in the enum. Per the cascade rule, every epoch completion must emit TRIAL_PROGRESS (among others).
    • Location: modules/common/src/rainbow/common/types/events.py — missing enum value
    • Impact: Orchestrator cannot observe per-epoch progress at the trial level.

    🎯 Final Canonical Flows To Keep

    This section is now the authoritative target taxonomy.

    Canonical design rule

    • Only keep events that belong to a real domain flow.
    • A real flow must answer four questions: who requests it, who executes/emits it, who consumes it, and what the terminal event is.
    • Default lifecycle for module jobs: *_REQUESTED → *_STARTED → *_PROGRESS → *_DONE.
    • Allowed exceptions:
    • *_ANOMALY_DETECTED as a live-safety side-channel.
    • HITL_* as a human approval flow.
    • Disallowed in the final canonical set: orphan *_READY, *_UPDATED, *_STOPPING, or single *_DONE events without a clear requester/executor/consumer contract.
    • Observability and telemetry are not business taxonomy. If we keep them, they must live in a separate observability spec, not in the canonical execution flow.
