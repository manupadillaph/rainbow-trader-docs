    Control Panel Frontend — Architecture Audit

    Date: 2026-03-19 Scope: modules/control-panel/frontend/src/ (280 source files, 73 directories) Stack: Next.js 16 (App Router, static export), React 19, SCSS Modules, lightweight-charts Objective: Identify duplication, naming inconsistencies, structural weaknesses, and misplaced responsibilities. Produce a prioritized remediation plan.

    1. Executive Summary

    The frontend is functional and well-typed (strict TypeScript, no any abuse). The code is readable and follows consistent patterns within each feature. However, it suffers from:

    • Duplicated utility logic across 3+ locations (query builders, int parsers).
    • 14 boilerplate client-page wrappers that could be a single factory.
    • Misplaced files (IndexPage.tsx likely dead, Panel.tsx orphaned at top level, backtest launch form inside trials/ directory).
    • No shared list-query hook — pagination/sorting/filtering boilerplate repeated in 4 list hooks.
    • Inconsistent path alias usage — @/* configured but used in exactly 1 file out of 280.
    • Near-zero test coverage — 3 test files for 280 source files.
    • price-scale/ page with ~300 lines of hardcoded test data (development artifact).

    Overall Architecture Score: 6.5/10 — Solid domain modeling and typing, but significant duplication and organizational debt.

    F-03 · HIGH — No Shared List Query Hook (Boilerplate in 4 Hooks)

    Affected files:

    • experiments/list/useExperimentsListPage.ts
    • trainings/list/useTrainingsListPage.ts
    • trials/list/useTrialsListPage.ts
    • backtests/list/useBacktestsListPage.ts

    All 4 hooks repeat the same pattern:

    1. Call useDashboardQuery() → extract page, pageSize, search, sort
    2. Extract entity-specific filters
    3. Build query object
    4. Fetch paginated data
    5. Return data + setSearch, setSort, goToPage setter functions

    Steps 1 and 5 are identical across all 4 hooks. A shared useDashboardListQuery<TSort>() hook would eliminate ~40 lines of boilerplate per hook.

    Fix: Create hooks/dashboard/useDashboardListQuery.ts that handles the common pagination/sorting/search state. Each entity hook calls it and adds entity-specific filters.

    Location: components/client-pages/ (14 files, each ~9 lines)

    Every file follows this exact pattern:

    "use client";
import dynamic from "next/dynamic";
export const XxxClient = dynamic(
    () => import("path").then((m) => m.XxxPage),

    // components/client-pages/createClientPage.ts
export const createClientPage = <T,>(
