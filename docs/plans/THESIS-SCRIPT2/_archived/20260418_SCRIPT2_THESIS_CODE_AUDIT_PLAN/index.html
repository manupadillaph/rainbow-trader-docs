    Fix 6 — Type Unification (101 classes → shared types)

    PRINCIPLE: Shared types live in reports.py. Each script uses the complete shared type and reads only what it needs. Script-local types are allowed ONLY for truly script-specific data (CLI args, computed views that compose shared types with local data). NO duplicated field sets across scripts.

    ONE NAME RULE: The shared type field names ARE the canonical names. ALL scripts MUST use the exact field names from TrialResult, ExperimentSummaryRecord, BacktestRunResult, and BacktestComparison. No short aliases, no mappings, no wrappers. If TrialResult says objective_value, every script says objective_value. If it says backtest_sharpe_ratio, every script says backtest_sharpe_ratio. Scripts adapt to the shared types, not the other way around.

    NO FALLBACKS RULE: If a field is expected, it MUST contain data. If it's None when it shouldn't be, that's an ERROR — raise an exception, don't silently compute a replacement or use an alternative field. No "or compute at runtime if needed", no "use X instead of Y". Data is there or the pipeline is broken and must be fixed.

    BUILDER PATTERN RULE: Every script-local type that extends a shared base type MUST have a builder/factory method that converts the base type into the extended type. Consistent naming: build_<type_name>(base, **extra_fields) → ExtendedType. Examples: build_completed_experiment_base(), build_trial_result_with_health(), build_completed_experiment_with_checkpoint(). All builders follow the same pattern: receive the base type + extra data, return the composed type. NO manual field-by-field copying scattered across script code.
