    1. Schema Conventions

    Applies to all tables:

    • Primary key column named id (int, autoincrement).
    • Foreign keys use snake_case with _id suffix (int) referencing the parent table's id.
    • Timestamps: created_at, updated_at; optional completed_at, scored_at, processed_at, etc., as needed.
    • Timestamp format: All timestamps use ISO 8601 format (YYYY-MM-DDTHH:MM:SSZ), e.g., "2025-01-15T10:30:00Z".
    • JSON payload columns use _json suffix when storing structured blobs.
    • Paths use path (not file_path) and hashes use _hash.

    2. Core Tables

    2.1 experiments

    Column Type Description
    id int (PK) Autoincrement
    name str Human-friendly name
    experiment_type str "trading" or "regime"
    status str pending, data_fetching, labeling, fat_building, training, scoring, completed, failed, stopping, stopped
    plan_hash str (nullable) Hash of ExperimentPlan
    experiment_config_json JSON (nullable) Full ExperimentPlan (planned)
    experiment_config_path str (nullable) Path to JSON plan (planned)
    n_trials_total int (nullable) Planned number of trials (planned)
    n_trials_completed int (nullable) Completed trials count (planned)
    n_trials_scored int (nullable) Scored trials count (planned)
    best_trial_id int (nullable, FK → trials.id) Best trial (planned)
    best_score float (nullable) Highest final_score (planned)
    created_at datetime When created
    updated_at datetime When updated

    Relationships: experiments.id is referenced by trials.experiment_id, events.experiment_id, parquet_files.experiment_id, labels_files.experiment_id, execution_plans.experiment_id, backtest_sessions.experiment_id.

    2.2 trials

    Column Type Description
    id int (PK) Autoincrement
    trial_uuid str (unique) UUID for the trial
    experiment_id int (FK → experiments.id) Owning experiment
    fat_parquet_id int (nullable, FK → parquet_files.id) FAT parquet for this trial
    small_parquet_id int (nullable, FK → parquet_files.id) SMALL parquet for this trial
    optuna_study_name str Optuna study name
    optuna_trial_number int Optuna trial number
    suffix str Suffix label (default: "base")
    phase int Optimization phase number
    window_size int Window size
    selected_columns JSON Selected columns and weights
    model_hyperparameters JSON Hyperparameters chosen by Optuna
    objective_value float (nullable) Objective metric value
    objective_value_raw float (nullable) Raw objective value before phase weighting
    last_epoch_objective_value float (nullable) Objective value from the latest completed validation epoch
    best_epoch_objective_value float (nullable) Best objective value across validation epochs seen during training
    stage1_result float (nullable) Stage 1 objective value (training_weights only)
    stage2_result float (nullable) Stage 2 objective value (training_weights only)
    stage3_return_pct float (nullable) Stage 3 return percentage
    stage3_total_pnl float (nullable) Stage 3 total PnL
    stage3_trades_count int (nullable) Stage 3 trades count
    training_metrics JSON (nullable) Training/validation metrics
    status str pending, running, completed, failed, pruned
    error_message str (nullable) Error details for failed trials
    started_at datetime (nullable) When the trial started
    completed_at datetime (nullable) When the trial completed
    created_at datetime When created
    updated_at datetime When updated

    Relationships: trials.id is referenced by events.trial_id, parquet_files.trial_id, backtest_trials.trial_id, backtest_sessions.trial_id, trades.trial_id.

    Constraints: Unique (experiment_id, suffix, phase, optuna_trial_number).

    2.3 events

    Column Type Description
    id int (PK) Autoincrement integer
    event_uuid str (unique, nullable) UUID for cross-system dedupe (optional now, recommended)
    event_type str e.g., "TRIAL_SCORED", "DATA_FETCH_DONE"
    experiment_id int (nullable, FK) For routing
    trial_id int (nullable, FK) For trial-specific events
    thread_id str (nullable) LangGraph thread to resume (default rainbow_main)
    payload JSON Event-specific data
    processed bool Whether Event Bridge has handled it
    processed_at datetime (nullable) When marked processed
    created_at datetime When event occurred (or inserted)
    updated_at datetime Touch-on-update

    Relationships: experiment_id → experiments.id; trial_id → trials.id.

    For event types and payloads, see EVENTS.md.

    3. Data Storage Tables

    3.1 parquet_files

    Column Type Description
    id int (PK) Autoincrement
    path str Absolute path to file
    category str "OHLCV", "indicator", "FAT", "SMALL", "labels"
    config_hash str (nullable) Hash of config used to build
    config_json JSON (nullable) Full config snapshot
    symbol str (nullable) For OHLCV/indicator files
    timeframe str (nullable) For OHLCV/indicator files
    experiment_id int (nullable, FK → experiments.id) Link to experiment
    trial_id int (nullable, FK → trials.id) Link to trial
    time_range_start datetime (nullable) Period start
    time_range_end datetime (nullable) Period end
    created_at datetime When created
    updated_at datetime When updated
    version int (nullable) File version

    3.2 labels_files

    Column Type Description
    id int (PK) Autoincrement
    experiment_id int (FK → experiments.id) Link to experiment
    path str Path to parquet file
    label_type str "trading" or "regime"
    labeling_config_hash str (nullable) Hash of labeling_config (planned)
    count_long int (nullable) Planned
    count_short int (nullable) Planned
    count_none int (nullable) Planned
    count_total int (nullable) Planned
    time_range_start datetime (nullable) Planned
    time_range_end datetime (nullable) Planned
    created_at datetime When created
    updated_at datetime When updated

    4. Backtest Tables

    4.1 backtest_trials

    Column Type Description
    id int (PK) Autoincrement
    trial_id int (FK → trials.id) Link to trial
    metrics JSON (nullable) Aggregated metrics (current)
    sharpe_ratio float (nullable) Planned convenience column
    profit_factor float (nullable) Planned convenience column
    max_drawdown float (nullable) Planned convenience column
    win_rate float (nullable) Planned convenience column
    total_trades int (nullable) Planned convenience column
    total_pnl float (nullable) Planned convenience column
    avg_trade_duration float (nullable) Planned convenience column (seconds)
    created_at datetime When created
    updated_at datetime When updated

    4.2 backtest_trades

    Column Type Description
    id int (PK) Autoincrement
    backtest_id int (FK → backtest_trials.id) Link to backtest
    side str "buy" or "sell"
    pnl float Profit/loss
    created_at datetime When created
    updated_at datetime When updated

    4.3 backtest_sessions

    Column Type Description
    id int (PK) Autoincrement
    backtest_session_uuid str (unique) UUID for the backtest session
    trial_id int (FK → trials.id) Link to trial
    experiment_id int (FK → experiments.id) Link to experiment
    backtest_type str mini, complete_post_optuna, manual
    status str running, completed, failed
    range_start datetime Backtest range start
    range_end datetime Backtest range end
    initial_equity float (nullable) Starting equity
    final_equity float (nullable) Final equity
    total_pnl float (nullable) Total PnL
    sharpe_ratio float (nullable) Sharpe ratio
    trades_count int (nullable) Number of trades
    win_rate float (nullable) Win rate
    started_at datetime (nullable) When the backtest started
    completed_at datetime (nullable) When the backtest finished
    created_at datetime When created
    updated_at datetime When updated

    Relationships: backtest_sessions.id is referenced by trades.backtest_session_id.

    5. Analysis Tables

    5.1 analyst_summaries

    Column Type Description
    id int (PK) Autoincrement
    experiment_id int (FK → experiments.id) Link to experiment
    trial_id int (nullable, FK → trials.id) Link to trial (for trial summaries)
    summary_type str "trial" or "experiment"
    summary_json JSON Structured summary
    regime_tags JSON (nullable) Regime tags for experiment
    recommendations_json JSON (nullable) Recommendations
    created_at datetime When created
    updated_at datetime When updated

    6. Execution Tables

    6.1 execution_plans

    Static definition of an execution plan. Runtime state is tracked in live_sessions.

    Column Type Description
    id int (PK) Autoincrement
    execution_plan_id str (unique) Human-readable ID (e.g., "exec_001")
    experiment_id int (FK → experiments.id) Link to experiment
    trial_id int (nullable, FK → trials.id) Link to trial
    target_symbol str Trading symbol
    target_timeframe str Trading timeframe
    execution_plan_json JSON Full ExecutionPlan config
    execution_plan_path str (nullable) Path to JSON file
    created_at datetime When created
    updated_at datetime When updated

    Note: mode (live/sim) is NOT stored here. It is derived from MasterExecutionPlan.slots which specifies whether this plan is in live_execution_plan_id or sim_execution_plan_ids.

    Note: status is NOT stored here. Runtime state is tracked in live_sessions.state.

    6.2 master_execution_plans

    Static definition of a master execution plan. Can have multiple plans, but only one ACTIVE at a time.

    Column Type Description
    id int (PK) Autoincrement
    master_execution_plan_id str (unique) Human-readable ID (e.g., "master_001")
    version_id int Increments on each update
    slots_json JSON Slots config: {symbol_TF: {live_execution_plan_id, sim_execution_plan_ids[]}}
    global_rules_json JSON (nullable) Global rules
    created_at datetime When created
    updated_at datetime When updated

    Note: Runtime state (ACTIVE, etc.) is NOT stored here. The active master is determined by which one has running live_sessions.

    Slots JSON structure:

    {
  "BTCUSDT_15m": {
    "live_execution_plan_id": "exec_001",
    "sim_execution_plan_ids": ["exec_002", "exec_003"]
  },
  "ETHUSDT_1h": {
    "live_execution_plan_id": "exec_004",
    "sim_execution_plan_ids": []
  }
}

    7. Live Trading Tables

    7.1 live_trades

    Column Type Description
    id int (PK) Autoincrement
    execution_plan_id int (nullable, FK → execution_plans.id) Link to execution plan
    live_session_id int (FK → live_sessions.id) Link to live session
    symbol str Trading symbol
    side str "buy" or "sell"
    entry_timestamp datetime Entry time
    exit_timestamp datetime (nullable) Exit time
    entry_price float Entry price
    exit_price float (nullable) Exit price
    size float Position size
    realized_pnl float (nullable) Realized profit/loss
    unrealized_pnl float (nullable) Unrealized profit/loss
    status str "open", "closed"
    exchange_order_id str Exchange order ID
    created_at datetime When created
    updated_at datetime When updated

    Note: Current ORM uses the unified trades table for both backtest and live runs.

    7.2 live_sessions

    Dynamic runtime state for each running execution. One LiveSession per running bot (live or sim).

    Column Type Description
    id int (PK) Autoincrement
    live_session_id str (unique) UUID auto-generated on start
    master_execution_plan_id str (FK → master_execution_plans.master_execution_plan_id) Which master plan
    execution_plan_id str (FK → execution_plans.execution_plan_id) Which execution plan
    slot str Slot identifier (e.g., "BTCUSDT_15m")
    mode str "live" or "sim" (derived from master at creation)
    state str "RUNNING", "PAUSED", "STOPPED"
    started_at datetime Live session start
    stopped_at datetime (nullable) Live session end
    total_trades int (nullable) Number of trades (updated on stop)
    total_pnl float (nullable) Total profit/loss (updated on stop)
    metrics_json JSON (nullable) Additional metrics
    created_at datetime When created
    updated_at datetime When updated

    Constraints: - Only ONE live_session with mode='live' and state='RUNNING' per slot - Multiple live_session with mode='sim' and state='RUNNING' allowed per slot

    Identification rules: - Live mode: unique per slot → can identify with slot + mode='live' - Sim mode: multiple per slot → MUST identify with live_session_id or execution_plan_id

    7.3 trades

    Unified trade records for both backtest and live trading.

    Column Type Description
    id int (PK) Autoincrement
    trial_id int (FK → trials.id) Trial used for this trade
    live_session_id int (nullable, FK → live_sessions.id) Live session linkage
    backtest_session_id int (nullable, FK → backtest_sessions.id) Backtest session linkage
    side str "LONG" or "SHORT"
    size float Position size
    entry_price float Entry price
    exit_price float (nullable) Exit price
    stop_loss float Stop loss price
    take_profit float Take profit price
    pnl float Profit/loss
    reason str (nullable) Close reason
    opened_at datetime Open timestamp
    closed_at datetime (nullable) Close timestamp
    is_open bool True if open, False if closed
    is_backtest bool True for backtest trades, False for live
    trailing_stop_distance float (nullable) Trailing stop distance
    trailing_stop_activation float (nullable) Trailing stop activation
    highest_price float (nullable) Highest price seen (LONG)
    lowest_price float (nullable) Lowest price seen (SHORT)
    created_at datetime When created
    updated_at datetime When updated

    Constraints:

    • Exactly one of live_session_id or backtest_session_id must be set
    • trial_id is required for all trades

    8. Regime Tables

    8.1 regimes

    Column Type Description
    id int (PK) Autoincrement
    experiment_id int (nullable, FK → experiments.id) Source experiment (if ML-based)
    symbol str Symbol
    timeframe str Timeframe
    start_timestamp datetime Period start
    end_timestamp datetime Period end
    regime_label str e.g., "bull", "bear", "range", "high_vol"
    confidence float (nullable) Confidence score
    created_at datetime When created
    updated_at datetime When updated

    9. LangGraph Tables

    9.1 Checkpointer Tables

    Managed by LangGraph Checkpointer:

    • checkpoints: Graph state checkpoints
    • checkpoint_blobs: Binary data for checkpoints
    • checkpoint_writes: Write operations log

    9.2 Store Tables

    Managed by LangGraph Store:

    • store_items: Key-value items for agent long-term memory
    • store_vectors: Vector embeddings for semantic search (if using vector store)
