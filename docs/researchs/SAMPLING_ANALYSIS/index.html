    Sampling & Class Weights Analysis

    Current Problem

    Model performs well on Nov 2024 - Jan 2025 (WR 85%) but degrades to 47% WR by Aug 2025.

    Root Cause

    Over-aggressive signal prediction due to class weight configuration.

    Current Settings

    {
  "training_none_ratio": 2.5,
  "validation_none_ratio": 3.5,
  "class_weights": "auto"  // Results in [0.5714, 1.0, 1.0]
}

    Formula

    none_weight = max(0.5, 1.0 / (1.0 + training_ratio * 0.3))
# With training_ratio = 2.5:
# none_weight = 0.5714

    Effect

    1. Sampling reduces NONEs: 83% → 71% of data
    2. Class weights penalize NONE less: 0.57 vs 1.0 for signals
    3. Model learns to over-predict signals: Finds patterns where none exist

    Why It Works Initially Then Fails

    Good Period (Nov 2024 - Jan 2025): - Trending market with clear signals - Over-prediction = catching real opportunities - High win rate because signals were genuine

    Bad Period (Feb 2025 - Aug 2025): - Choppy/ranging market - Over-prediction = false signals in noise - Low win rate because trading on randomness

    Proposed Solutions

    Option 1: Increase NONE Weight (Recommended)

    Change:

    # Current formula
none_weight = max(0.5, 1.0 / (1.0 + training_ratio * 0.3))

# Proposed formula
none_weight = max(0.7, 1.0 / (1.0 + training_ratio * 0.2))
# With training_ratio = 2.5:
# none_weight = max(0.7, 1.0 / 1.5) = max(0.7, 0.667) = 0.7

# Result: [0.7, 1.0, 1.0] instead of [0.5714, 1.0, 1.0]

    Pros: - Simple one-line change - Makes model more conservative - Better precision, fewer false signals

    Cons: - May reduce recall (miss some real signals) - Need to retrain to see effect

    Option 2: INCREASE Sampling Ratio (Keep More NONEs)

    Change:

    {
  "training_none_ratio": 3.5,  // Was 2.5 (INCREASE to keep MORE NONEs)
  "validation_none_ratio": 4.5  // Was 3.5 (INCREASE to keep MORE NONEs)
}

    Effect: - Keeps MORE NONEs in dataset - 71% → 78% NONE (vs current 71%) - Model sees MORE "do nothing" examples

    Calculation:

    # With 1000 LONG + 1000 SHORT = 2000 signals
# Current ratio 2.5: 2000 * 2.5 = 5000 NONEs → 71% NONE
# New ratio 3.5: 2000 * 3.5 = 7000 NONEs → 78% NONE

    Pros: - Model learns when NOT to trade (more conservative) - Reduces false signals in choppy markets - May improve precision

    Cons: - Requires re-labeling with new sampling - Longer training time (more data) - May reduce recall (miss some real signals) - May bias towards inaction

    Option 3: Manual Class Weights

    Change:

    {
  "class_weights": [0.8, 1.0, 1.0]  // Manual instead of "auto"
}

    Pros: - Direct control - Easy to experiment - No code changes needed

    Cons: - Requires experimentation - Optimal values unclear - May need tuning per experiment

    Option 4: Focal Loss

    Replace cross-entropy loss with Focal Loss:

    # Focal Loss automatically down-weights easy examples
# Focuses learning on hard-to-classify cases
# Good for imbalanced datasets

focal_loss = FocalLoss(alpha=[0.7, 1.0, 1.0], gamma=2.0)

    Pros: - State-of-art for imbalanced classification - Adaptive weighting during training - Better than fixed class weights

    Cons: - Requires implementing new loss function - More hyperparameters to tune - Harder to debug

    Recommendation

    Start with Option 1 (increase NONE weight to 0.7):

    1. Quick test: Change formula and retrain one trial
    2. Validate: Check if precision improves without losing recall
    3. Compare: Stage2 F1 should increase from 11% to >30%
    4. Backtest: Stage3 should be more stable across time

    If that doesn't work well enough, combine with Option 2 (increase sampling ratio to 3.5 to keep more NONE examples).

    Implementation

    Step 1: Modify Class Weight Calculation

    File: modules/model-trainer/src/rainbow/model_trainer/helpers/data_preparation.py

    def calculate_class_weights(*, sampling_config: SamplingConfig) -> torch.Tensor:
    training_ratio = float(sampling_config.training_none_ratio)
    if training_ratio < 0:
        raise ValueError(...)

    # OLD: none_weight = max(0.5, 1.0 / (1.0 + training_ratio * 0.3))
    # NEW: Make NONE weight higher (less penalization of NONE class)
    none_weight = max(0.7, 1.0 / (1.0 + training_ratio * 0.2))

    weights = torch.tensor([none_weight, 1.0, 1.0], dtype=torch.float32)
    return weights

    Step 2: Test on Single Trial

    # Run one trial with new weights
# Compare metrics:
# - Stage1 (training): expect accuracy ~93% (vs current 95.8%)
# - Stage2 (validation): expect F1 >30% (vs current 11%)
# - Stage3 (backtest): expect more stable WR across time

    Step 3: Full Optuna Study

    If initial trial shows improvement, run full study with new weights.

    Metrics to Watch

    Before (Current)

    • Stage1: 95.8% acc, 92.9% F1 ← Overfitting
    • Stage2: 70.4% acc, 11.2% F1 ← Terrible generalization
    • Stage3: WR 85% → 47% ← Degradation over time

    After (Expected)

    • Stage1: 92-94% acc, 90% F1 ← Less overfitting
    • Stage2: 75-80% acc, 35-45% F1 ← Better generalization
    • Stage3: WR 75% → 65% ← More stable, conservative

    ==================================================================================================== REGIME VARIABLE EXPLANATIONS ====================================================================================================

    ── VARIABLE 1: TREND (based on ADX — Average Directional Index) ── ADX measures trend STRENGTH (0-100), not direction. Threshold: ADX >= 25.0 = TRENDING | ADX <= 20.0 = RANGING

    TRENDING — Strong directional movement. Price moves decisively in one direction. ADX >= 25. Market has clear momentum. Good for trend-following strategies. Example: sustained rally or sustained sell-off.

    NEUTRAL — Moderate trend. ADX between 20-25. Market is transitioning — could be starting a trend or losing one. Ambiguous environment.

    RANGING — No clear trend. ADX <= 20. Price oscillates sideways within a range. Good for mean-reversion strategies. Bad for trend-following. Example: consolidation phase, low-conviction market.

    ── VARIABLE 2: VOLATILITY (based on ATR/Price ratio vs rolling average) ── ATR ratio = ATR / Close price (percentage volatility). Compared to 2016-bar rolling average (~1 week of 5m data). Threshold: > 1.2x avg = HIGH | < 0.8x avg = LOW

    HIGH_VOL — Volatility is 20%+ above its recent average. Big candles, wide ranges. Often during: news events, liquidation cascades, breakouts, panic. Risk: stop-losses get hit more. Reward: bigger moves to capture.

    NORMAL_VOL— Volatility is within ±20% of its average. Typical market conditions. Most predictable environment for models. Patterns hold better.

    LOW_VOL — Volatility is 20%+ below its recent average. Small candles, tight range. Often before: breakouts (volatility squeeze). During: low-liquidity periods. Risk: false signals from noise. Reward: tighter stops possible.

    ── VARIABLE 3: DIRECTION (based on +DI/-DI crossover + EMA fast/slow alignment) ── +DI > -DI AND EMA(14) > EMA(50) = BULLISH -DI > +DI AND EMA(14) < EMA(50) = BEARISH Otherwise = NEUTRAL (DI and EMA disagree)

    BULLISH — Both indicators confirm upward pressure. Buyers dominate. +DI > -DI (more upward directional movement) AND fast EMA above slow EMA (short-term price above long-term).

    NEUTRAL — Mixed signals. DI says one thing, EMA says another. Transitional phase — often at trend reversals or in choppy conditions. Model predictions are least reliable here.

    BEARISH — Both indicators confirm downward pressure. Sellers dominate. -DI > +DI (more downward directional movement) AND fast EMA below slow EMA (short-term price below long-term).
