    Control Panel Components Design

    Status: In Progress
    Started: 2026-02-18
    Priority: High

    Overview

    Design document for the 3 reusable components that will replace ~12-15 legacy components in the Control Panel.

    Component Names: - Table - Generic table for all entities - ContextCard - Context card for all entity types - ListLayout - Page layout wrapper for all list pages

    1. Table Component

    Purpose

    ONE table component that works for ALL entities: Trials, Trainings, Experiments, Backtests.

    Key Features

    • Variant system: full/compact/minimal
    • ALL columns sortable (except actions/complex cells)
    • Per-table density: compact/comfortable
    • Pagination built-in
    • Context-aware routing
    • Type-safe column configuration

    Props Interface

    interface UnifiedEntityTableProps<TEntity, TSort extends string> {
  // Data
  entity: "trials" | "trainings" | "experiments" | "backtests";
  rows: TEntity[];

  // Pagination
  page: number;
  pageSize: number;
  totalPages: number;
  totalItems: number;
  onPrevious: () => void;
  onNext: () => void;

  // Sorting - ALL columns
  sort: TSort;
  onSort: (value: TSort) => void;

  // Display
  variant: "full" | "compact" | "minimal";

  // Context (for routing)
  context?: {
    experimentId?: number;
    studyId?: number;
    rangeStart?: string;
    rangeEnd?: string;
  };

  // Optional features
  launchForm?: UseBacktestLaunchFormResult; // For trials
  showDensityToggle?: boolean; // Default true for full variant
}

    Column Configuration System

    Each entity has a column registry that defines ALL possible columns:

    // columns/trialColumns.ts
export const TRIAL_COLUMNS = {
  id: {
    key: "id",
    label: "ID",
    sortKey: "id",
    sortable: true,
    render: (trial: TrialSummary) => <span className="mono">{trial.id}</span>,
  },
  optunaNumber: {
    key: "optunaNumber",
    label: "Optuna #",
    sortKey: "optuna_number",
    sortable: true,
    render: (trial: TrialSummary) => <span>#{trial.optuna_trial_number ?? "n/a"}</span>,
  },
  status: {
    key: "status",
    label: "Status",
    sortKey: "status",
    sortable: true,
    render: (trial: TrialSummary) => <StatusBadge value={trial.status} />,
  },
  objective: {
    key: "objective",
    label: "Objective",
    sortKey: "objective",
    sortable: true,
    render: (trial: TrialSummary) => formatFloatValue(trial.objective_value),
  },
  stages: {
    key: "stages",
    label: "Stages (1/2/3)",
    sortKey: "stage3_result", // Sort by stage 3
    sortable: true,
    render: (trial: TrialSummary) => <StagesCell trial={trial} />,
  },
  metrics: {
    key: "metrics",
    label: "Eq/PnL/Trades",
    sortKey: "stage3_total_pnl",
    sortable: true,
    render: (trial: TrialSummary) => <MetricsCell trial={trial} />,
  },
  phaseStudy: {
    key: "phaseStudy",
    label: "Phase/Study",
    sortKey: "phase",
    sortable: true,
    render: (trial: TrialSummary, context) => <PhaseStudyCell trial={trial} context={context} />,
  },
  bestFlags: {
    key: "bestFlags",
    label: "Best Flags",
    sortKey: null, // Not sortable - composite field
    sortable: false,
    render: (trial: TrialSummary) => (
      <BestFlags
        isBestForExperiment={trial.is_best_for_experiment}
        isBestForStudy={trial.is_best_for_study}
      />
    ),
  },
  actions: {
    key: "actions",
    label: "Actions",
    sortKey: null,
    sortable: false,
    render: (trial: TrialSummary, context, launchForm) => (
      <ActionsCell trial={trial} context={context} launchForm={launchForm} />
    ),
  },
} as const;

export type TrialColumnKey = keyof typeof TRIAL_COLUMNS;

    2. UnifiedContextCard Component

    Purpose

    ONE context card component that shows entity summaries (Experiment, Training, Trial) across all pages.

    Key Features

    • Multi-entity support: experiment/training/trial
    • Variant system: full/compact
    • Configurable actions: all/subset/custom
    • Metrics display: Flexible metric rows
    • Type-safe props

    Props Interface

    interface UnifiedContextCardProps {
  // Entity type and data
  entity: "experiment" | "training" | "trial";
  data: ExperimentDetail | TrainingSummary | TrialSummary;

  // Display
  variant: "full" | "compact";

  // Actions
  actions?: "all" | "subset" | CustomAction[];
  showRequirements?: boolean; // For experiments only

  // Context for routing
  experimentId?: number;
  studyId?: number;
}

interface CustomAction {
  label: string;
  href: string;
}

    Usage Examples

    // Experiment context in trials page
<UnifiedContextCard
  entity="experiment"
  data={experiment}
  variant="full"
  actions="all"
  showRequirements={false}
/>

// Experiment context in experiment detail
<UnifiedContextCard
  entity="experiment"
  data={experiment}
  variant="full"
  actions="all"
  showRequirements={true}
/>

// Training context in trials page
<UnifiedContextCard
  entity="training"
  data={training}
  variant="compact"
  actions="subset"
  experimentId={experimentId}
/>

    3. EntityListPageLayout Component

    Purpose

    ONE page wrapper that handles the common list page pattern: filters → context → table → pagination.

    Key Features

    • Clear visual hierarchy: Filters visually connected to tables
    • Flexible sections: Context, launch panel, table
    • Consistent spacing
    • Loading/error states

    Props Interface

    interface EntityListPageLayoutProps {
  // Page metadata
  title: string;
  subtitle: string;

  // Sections (all optional)
  filters?: ReactNode;
  contextCard?: ReactNode;
  launchPanel?: ReactNode;
  table: ReactNode;

  // States
  loading?: boolean;
  error?: string | null;

  // Additional content
  children?: ReactNode;
}

    Usage Example

    <EntityListPageLayout
  title="Trials"
  subtitle="Review trial outcomes and launch new backtests from selected models."
  filters={
    <TrialsFilters
      experimentFilter={experimentFilter}
      statusFilter={statusFilter}
      // ...
    />
  }
  contextCard={
    contextExperiment.data ? (
      <UnifiedContextCard
        entity="experiment"
        data={contextExperiment.data}
        variant="full"
        actions="all"
      />
    ) : null
  }
  launchPanel={
    <TrialBacktestLaunchPanel
      contextExperimentId={contextExperimentId}
      launchForm={launchForm}
    />
  }
  table={
    <UnifiedEntityTable
      entity="trials"
      rows={trials.data?.items ?? []}
      variant="full"
      // ...
    />
  }
  loading={trials.loading}
  error={trials.error}
/>

    Implementation Order

    1. Phase 1: Create column definitions for all entities
    2. Phase 2: Implement UnifiedEntityTable
    3. Phase 3: Implement UnifiedContextCard
    4. Phase 4: Implement EntityListPageLayout
    5. Phase 5: Replace all legacy components (all entities at once)
    6. Phase 6: Test all pages
    7. Phase 7: Delete all legacy components

    Sorting Verification

    For EACH page above, verify: - [ ] ALL columns are sortable (except actions/complex cells) - [ ] Clicking column header changes sort - [ ] Sort direction indicator shows correctly - [ ] Data actually sorts correctly

    Density Verification

    • [ ] Density toggle appears on full variant tables
    • [ ] Density toggle does NOT appear on compact/minimal tables
    • [ ] Changing density persists in localStorage
    • [ ] Each table has independent density setting

    Visual Verification

    • [ ] Filters are visually connected to tables
    • [ ] Clear hierarchy: filters → context → table
    • [ ] No confusion about what filters affect what
    • [ ] Consistent spacing and alignment

    Success Criteria

    ✅ Code Quality: - No duplicated table components - No duplicated context components - Clean, modular, reusable code - Type-safe throughout - No legacy code remaining

    ✅ Functionality: - ALL columns sortable (where appropriate) - Pagination works on all pages - Density toggle works and persists - Context routing works correctly - All actions work (Detail, Backtests, etc.)

    ✅ User Experience: - Clear visual hierarchy - Obvious filter → table connection - Consistent behavior across all pages - No visual regressions

    ✅ Maintainability: - Easy to add new columns - Easy to add new entities - Easy to modify variants - Well-documented code
