    Indicators

    Overview

    Each indicator generates specific columns with defined validation rules. Indicators also define:

    • column_outputs_generated: The columns the indicator produces
    • relative_meta: How to relativize each column (ClassVar defined in indicator class)
    • normalization_meta: How to normalize each column after relativization (ClassVar defined in indicator class)

    1. RSI (Relative Strength Index)

    Source Code:

    • Class definition: @/home/manuelpadilla/sources/reposUbuntu/PERSONALS/rainbow-trader/modules/core/src/rainbow/core/types/composed/indicators.py:493-543
    • Config class: @/home/manuelpadilla/sources/reposUbuntu/PERSONALS/rainbow-trader/modules/common/src/rainbow/common/types/configs/indicators/rsi_config.py
    • YAML config: @/home/manuelpadilla/sources/reposUbuntu/PERSONALS/rainbow-trader/config/indicators/rsi.yaml

    Type: IndicatorType.RSI
    Feature Group: MOMENTUM

    Generated Columns

    • rsi

    Validation Rules

    Column Min Max Tolerance Allow NaN Allow Inf Max Gaps
    rsi 0.0 100.0 0.0 false false 0

    Relative Meta (ClassVar)

    relative_meta = {
    "rsi": RelativeIndicatorMeta(method=RelativeMethod.DIVIDE_BY_100)
}

    Rationale: RSI is already 0-100, divide by 100 to get 0-1 range.

    Normalization Meta (ClassVar)

    normalization_meta = {
    "rsi_rel": NormalizationMethod.NONE
}

    Rationale: After relativization, RSI is 0-1 - no normalization needed.

    Configuration (YAML)

    rsi:
  period: 14

    Warm-up Periods

    • Formula: period + 1
    • Value: 15 rows

    Each indicator defines its own relative_meta (ClassVar) specifying:

    • method: How to calculate relative value
    • DIVIDE_BY_INPUT: Divide by an input column (e.g., ema/close)
    • DIVIDE_BY_100: Divide by 100 (for 0-100 indicators like RSI)
    • ROLLING_RATIO: Divide by rolling mean of self (for cumulative indicators like OBV)
    • BAND_POSITION: Calculate position within bands (Bollinger Bands)
    • NONE: No relativization needed
    • base_input_name: For DIVIDE_BY_INPUT method, which input to use as divisor

    Generated Columns (Added)

    For each computed column, adds a relativized version with suffix:

    • Example: rsi → rsi_rel, ema → ema_rel

    Normalization Methods

    Method Formula Use Case
    z_score (x - mean) / std Standard normalization, assumes normal distribution
    robust (x - median) / IQR Robust to outliers, uses median and interquartile range
    min_max (x - min) / (max - min) Scale to [0, 1] range
    none x (no change) Skip normalization for this column
