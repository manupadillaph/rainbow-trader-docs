    Introduction: Why So Many Metrics?

    When training a trading model, you'll encounter dozens of numbers. It's overwhelming at first. But here's the key insight:

    Different metrics measure different aspects of model quality.

    Think of it like evaluating a car:

    • Speed (0-100 km/h time) → How fast can it go?
    • Fuel efficiency (km/L) → How economical is it?
    • Safety rating (1-5 stars) → How safe is it?
    • Reliability (years without repair) → How dependable is it?

    A car with amazing speed but terrible safety isn't "the best car" - it depends on what you need. Same with ML models.

    For trading, we care about:

    1. Does the model predict correctly? (ML metrics)
    2. Does it make money? (Trading metrics)
    3. Is the performance consistent? (Risk-adjusted metrics)

    Classification Metrics (ML Fundamentals)

    Our model predicts one of three classes:

    • HOLD (0): Do nothing
    • LONG (1): Buy / Go long
    • SHORT (2): Sell / Go short

    Accuracy

    What it measures: The percentage of predictions that were correct.

    Accuracy = Correct Predictions / Total Predictions

    Example:

    • Model made 100 predictions
    • 65 were correct (predicted LONG when price went up, etc.)
    • Accuracy = 65% = 0.65

    Intuition: "If the model says something, how often is it right overall?"

    ⚠️ The Problem with Accuracy:

    Imagine 90% of the time the market is "HOLD". A dumb model that always predicts HOLD would have 90% accuracy! But it's useless for trading.

    This is called the class imbalance problem. That's why we need other metrics.

    When to use: As a quick sanity check, but never as the only metric.

    Loss

    What it measures: How "wrong" the model's confidence scores are. Lower is better.

    Intuition: Think of it as "regret" or "error magnitude".

    If model predicts:  LONG with 90% confidence
But reality was:    SHORT

→ High loss! The model was very confident AND very wrong.

    Types of loss:

    • Training loss: Error on data the model learned from
    • Validation loss (val_loss): Error on unseen data ← This one matters more!

    Key insight:

    • If training_loss is low but val_loss is high → Overfitting (memorized training data)
    • Both should be similar for a good model

    Range: Technically 0 to infinity, but in practice:

    • < 0.5: Excellent
    • 0.5 - 1.0: Good
    • 1.0 - 1.5: Acceptable

    • 1.5: Poor

    Precision

    What it measures: When the model predicts a positive class (LONG or SHORT), how often is it actually correct?

    Precision = True Positives / (True Positives + False Positives)

    Trading intuition: "Of all the trades the model recommended, how many were actually good?"

    Example:

    • Model predicted LONG 20 times
    • 14 times the price actually went up (True Positives)
    • 6 times the price went down (False Positives)
    • Precision = 14/20 = 70%

    High precision means: Fewer false alarms. When the model says "trade", you can trust it more.

    Low precision means: Many false signals. The model recommends trades that don't work out.

    Use case: You want high precision when the cost of a wrong trade is high (big position sizes, high fees).

    Recall

    What it measures: Of all the actual opportunities in the market, how many did the model catch?

    Recall = True Positives / (True Positives + False Negatives)

    Trading intuition: "Of all the good trading opportunities that existed, how many did the model identify?"

    Example:

    • There were 30 actual LONG opportunities in the data
    • Model correctly predicted 18 of them
    • Recall = 18/30 = 60%

    High recall means: The model doesn't miss opportunities. It catches most of the good trades.

    Low recall means: The model is too conservative. It misses many good trades.

    Use case: You want high recall when missing opportunities is costly (trending markets, limited time windows).

    F1 Score

    What it measures: The harmonic mean of Precision and Recall. A balanced view.

    F1 = 2 × (Precision × Recall) / (Precision + Recall)

    Why harmonic mean? Because it penalizes extremes. If precision is 100% but recall is 0%, F1 = 0 (not 50%).

    Trading intuition: "Is the model both careful (precision) AND aggressive enough to catch opportunities (recall)?"

    Example calculations:

    Scenario Precision Recall F1 Score
    Balanced 70% 70% 70%
    Conservative 90% 30% 45%
    Aggressive 30% 90% 45%
    Perfect 100% 100% 100%

    Range: 0 to 1 (or 0% to 100%)

    • 0.5+: Decent
    • 0.6+: Good
    • 0.7+: Very good
    • 0.8+: Excellent (rare in trading)

    F1 Macro: The average F1 across all classes (HOLD, LONG, SHORT). This is what we typically use because it treats each class equally.

    Trading-Specific Metrics

    ML metrics tell us if the model predicts correctly, but trading metrics tell us if it makes money.

    PnL (Profit and Loss)

    What it measures: Total money gained or lost.

    PnL = Sum of all trade profits - Sum of all trade losses

    Units: Dollars (or whatever currency)

    Example:

    • Started with $10,000
    • Made 5 winning trades: +$200, +$150, +$300, +$100, +$250 = +$1,000
    • Made 3 losing trades: -$100, -$200, -$150 = -$450
    • PnL = +$550

    Intuition: The bottom line. Did we make or lose money?

    ⚠️ Limitation: PnL alone doesn't tell you:

    • How risky were the trades?
    • Was it one lucky trade or consistent?
    • What was the starting capital?

    Return Percentage

    What it measures: PnL as a percentage of starting capital.

    Return % = (PnL / Starting Capital) × 100

    Example:

    • PnL = +$550
    • Starting Capital = $10,000
    • Return = 5.5%

    Why it matters: Allows comparison across different account sizes.

    Making $1,000 on a $10,000 account (10%) is more impressive than making $1,000 on a $1,000,000 account (0.1%).

    Win Rate

    What it measures: Percentage of trades that were profitable.

    Win Rate = Winning Trades / Total Trades

    Example:

    • 87 total trades
    • 48 were profitable
    • Win Rate = 48/87 = 55.2%

    Intuition: "Out of every 10 trades, how many make money?"

    Important ranges:

    • < 40%: Poor (losing more often than winning)
    • 40-50%: Acceptable (if winners are bigger than losers)
    • 50-60%: Good
    • 60%+: Excellent

    ⚠️ Win Rate Trap:

    A 90% win rate sounds amazing, but consider:

    • 9 winning trades of $10 each = +$90
    • 1 losing trade of $100 = -$100
    • Net PnL = -$10 😱

    This is why we also need the risk/reward ratio and Sharpe ratio.

    Sharpe Ratio

    What it measures: Risk-adjusted return. How much return per unit of risk?

    Sharpe Ratio = (Average Return - Risk-Free Rate) / Standard Deviation of Returns

    In practice for trading:

    Sharpe Ratio ≈ Mean(daily returns) / StdDev(daily returns) × √252

    Intuition: "Am I being compensated enough for the risk I'm taking?"

    Example:

    • Strategy A: +20% return with 40% volatility → Sharpe ≈ 0.5
    • Strategy B: +10% return with 5% volatility → Sharpe ≈ 2.0

    Strategy B is actually BETTER on a risk-adjusted basis, even though raw return is lower.

    Ranges: | Sharpe Ratio | Interpretation | |--------------|----------------| | < 0 | Losing money | | 0 - 0.5 | Poor | | 0.5 - 1.0 | Acceptable | | 1.0 - 2.0 | Good | | 2.0 - 3.0 | Very Good | | > 3.0 | Excellent (suspicious - check for overfitting) |

    Why it's crucial:

    • A strategy with Sharpe > 1 is generally worth trading
    • A strategy with Sharpe < 0.5 is probably not worth the stress

    The Three Stages: What They Mean

    Rainbow Trader evaluates models in three stages:

    Stage 1: Training Metrics

    Data used: The data the model learned from (training set).

    What it tells you: "Can the model learn patterns at all?"

    ⚠️ Warning: Good Stage 1 metrics don't guarantee good real performance. The model might be memorizing, not learning.

    What to look for:

    • val_loss should be decreasing
    • Accuracy should be reasonable (not 33% which is random for 3 classes)

    Stage 2: Validation Metrics

    Data used: Data the model has NEVER seen during training.

    What it tells you: "Can the model generalize to new data?"

    This is the most important ML metric stage!

    What to look for:

    • val_loss similar to Stage 1 (if much higher = overfitting)
    • F1 score as high as possible
    • Accuracy meaningfully above random (>40% for 3 classes)

    Stage 3: Backtest Metrics (Mini-Backtest)

    Data used: A small "tail" of the validation period, simulating real trading.

    What it tells you: "Would this actually make money?"

    This is what matters most for real trading!

    What to look for:

    • Positive PnL (obviously)
    • Win Rate > 40% (consistency)
    • Sharpe Ratio > 0.5 (risk-adjusted performance)
    • Reasonable trade count (not just 2 lucky trades)

    Decision Framework: How to Pick the Best Model

    Step 1: Filter by Trading Metrics (Stage 3)

    Eliminate any trial that:

    • ❌ Has negative PnL
    • ❌ Has less than 20 trades (not statistically significant)
    • ❌ Has Sharpe < 0

    Step 2: Rank by Risk-Adjusted Return

    Among surviving trials, prioritize:

    1. Sharpe Ratio (>1 is ideal)
    2. Win Rate (>50% is ideal)
    3. PnL / Return %

    Step 3: Check for Overfitting

    Compare Stage 1 and Stage 2 metrics:

    • If Stage 2 is MUCH worse → model is overfitting
    • If Stage 2 is similar → model generalizes well

    Step 4: Consider the Phase

    Higher phases (4, 5) used more data for training:

    • Phase 1: 15% of data
    • Phase 2: 20%
    • Phase 3: 25%
    • Phase 4: 33%
    • Phase 5: 50%

    Models from higher phases are generally more robust (but might take longer to train).

    Common Pitfalls

    1. "High Accuracy = Good Model"

    Wrong! Accuracy is misleading with imbalanced classes. A model predicting HOLD 90% of the time in a HOLD-dominated market has high accuracy but is useless.

    Solution: Look at F1 score and trading metrics instead.

    2. "Lowest Loss = Best Model"

    Wrong! Loss is about prediction confidence, not trading profitability. A model can have low loss but still lose money.

    Solution: Stage 3 PnL is the ultimate judge.

    3. "More Trades = More Profit"

    Wrong! Quality over quantity. 100 mediocre trades with 45% win rate will lose money to fees.

    Solution: Look at Sharpe ratio and average trade quality.

    4. "Stage 1 Metrics Matter"

    Wrong! Stage 1 uses training data. The model has seen it before. It's like testing a student with the same questions they studied.

    Solution: Focus on Stage 2 (validation) and Stage 3 (backtest) metrics.

    5. "Ignore Sharpe if PnL is Positive"

    Wrong! A positive PnL with negative Sharpe means you made money by luck, not skill. It won't repeat.

    Solution: Require Sharpe > 0.5 minimum.

    Summary Cheat Sheet

    METRICS TO PRIORITIZE (in order):

1. Stage 3 PnL         → Must be POSITIVE
2. Stage 3 Sharpe      → Want > 1.0 (minimum 0.5)
3. Stage 3 Win Rate    → Want > 50% (minimum 40%)
4. Stage 2 F1          → Higher = better generalization
5. Objective Value     → Composite score (useful for comparison)

METRICS TO BE CAREFUL WITH:

- Accuracy      → Misleading with imbalanced classes
- Stage 1 *     → Overfitting risk (training data)
- Raw PnL       → Without context of risk taken

RED FLAGS:

❌ Stage 2 loss >> Stage 1 loss (overfitting)
❌ Very few trades (< 20) - not statistically significant
❌ Negative Sharpe with positive PnL (lucky, not skilled)
❌ 100% win rate (definitely overfitted or too few trades)

    Conclusion

    Metrics are tools, not answers. The "best" model depends on your goals:

    • Maximum returns: Prioritize PnL and Return %
    • Sleep at night: Prioritize Win Rate and Sharpe
    • Robustness: Prioritize Stage 2 F1 and use higher phases

    For Rainbow Trader specifically, Trial 85 emerged as the best candidate because it excels where it matters most: making money consistently with acceptable risk.

    Remember: ML metrics (accuracy, F1) tell you if the model is "smart", but trading metrics (PnL, Sharpe) tell you if it makes money. In the end, making money is what matters.
